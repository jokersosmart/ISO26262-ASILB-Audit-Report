# FMEDA 深度稽核分析報告：FailureRateCalc

## 1. Sheet 概述

- **Sheet 名稱**: FailureRateCalc
- **在 FMEDA 報告中的位置**: 第 13 個 sheet
- **維度**: B1:AR43 (43 行 × 44 列)
- **角色與目的**: 此工作表是整個 FMEDA 分析的數據基石，其核心目的在於根據國際標準 IEC TR 62380，為積體電路（IC）提供一個標準化、可追溯的失效率（Failure Rate）計算框架。它將晶片設計、製程、封裝、以及預期操作環境等多維度參數，整合進一個數學模型，最終產出一個量化的失效率值（以 FIT 為單位，即每十億小時的失效次數）。這個值是後續所有安全分析（如 SPFM, LFM, PMHF 計算）的起點，其準確性直接決定了整個功能安全分析的有效性。

## 2. 內容結構分析

- **欄位標頭的含義與組織方式**: 標頭分散在工作表的各個區域，用於標示不同的計算模塊和參數表。例如，AF1:AR1 區域定義了與技術、材料相關的參數類型，而其他部分則散佈著如「Lithography Process」、「Average Transistor Size」等具體計算參數的標籤。這種組織方式是為了將 IEC TR 62380 標準中的表格和公式直接對應到 Excel 的計算環境中。
- **合併儲存格的用途**: 合併儲存格（如 F3:P3, D19:F19, H19:X19）主要用於創建標題或對某一區塊的計算進行總結性說明。例如，B3 單元格內的「IC failure rate calculation according to IEC TR 62380 (2004-08)」就是一個橫跨多列的標題，明確指出了此表的計算依據。
- **資料的組織邏輯**: 資料以模塊化的方式組織，每個模塊對應 IEC TR 62380 標準中的一個特定計算部分。例如，一部分區域用於輸入晶片的基本參數（如電晶體數量、製程節點），另一部分用於定義操作環境的溫度曲線，還有一部分則引用標準中的查找表（如材料的熱膨脹係數）。整個工作表的邏輯是引導使用者逐步輸入所有必要參數，最終在指定單元格中自動計算出總失效率。

## 3. 圖片分析

此工作表中包含三張關鍵圖片，它們並非裝飾性元素，而是 IEC TR 62380 標準的核心內容截圖，為表格中的計算提供了直接的理論和數據依據。

### 圖片 1 (sheet_13_FailureRateCalc_img_2.png)
- **圖片內容描述**: 該圖片展示了 IEC TR 62380 中用於計算 IC 失效率的完整數學模型。公式將總失效率 λ 分解為三個主要部分：晶粒失效率 (λ_die)、封裝失效率 (λ_package) 和電氣過應力失效率 (λ_overstress)。
- **代表意義**: 這是整個 `FailureRateCalc` 工作表的計算核心。它定義了所有輸入參數（如電晶體數量 N、溫度因子 π_t、封裝類型 λ₃ 等）如何組合在一起，形成最終的失效率預測。
- **與 ISO 26262 / IEC 61508 的對應關係**: ISO 26262 Part 11 附錄 D 提供了關於半導體失效率估算的指導，並引用了 IEC 62380 作為公認的行業方法之一。此公式是實現該指導的具體實踐。
- **對稽核的重要性**: **極高**。稽核員會逐項核對工作表中的 Excel 公式是否**完全且正確地**實現了這個數學模型。任何對公式的簡化、修改或錯誤實現都將是嚴重的稽核發現點。

### 圖片 2 (sheet_13_FailureRateCalc_img_0.png)
- **圖片內容描述**: 該圖片是 IEC TR 62380 中的「NECESSARY INFORMATION」表格，詳細列出了執行失效率計算所需的所有輸入參數及其定義、符號和參考來源（例如，「See Table 16」）。
- **代表意義**: 這張圖是使用者輸入數據的「操作手冊」。它告訴使用者需要收集哪些資料（如環境溫度、工作時間比率、電晶體數量等）才能完成計算。
- **與 ISO 26262 / IEC 61508 的對應關係**: 這對應了 ISO 26262 中對假設（Assumptions）和證據（Evidence）的要求。FMEDA 的所有輸入都必須有據可查，這張表格就是數據來源的頂層索引。
- **對稽核的重要性**: **高**。稽核員會用這張表作為參考，檢查 FMEDA 報告是否提供了所有必要參數的輸入值及其來源依據。任何缺失的參數或無法追溯來源的數據都會被質疑。

### 圖片 3 (sheet_13_FailureRateCalc_img_1.png)
- **圖片內容描述**: 該圖片是 IEC TR 62380 的 Table 16，提供了不同類型和製程的半導體元件的基礎失效率（λ₁ 和 λ₂）。例如，它明確列出了 ASIC（Standard Cell/Full Custom）的 λ₁ 為 1.2×10⁻⁵ FIT/gate。
- **代表意義**: 這是失效率計算中的一個關鍵**查照表 (Look-up Table)**。設計團隊需要根據其產品的具體類型（如 ASIC、FPGA、記憶體）從這個表格中選取正確的基礎失效率值。
- **與 ISO 26262 / IEC 61508 的對應關係**: 這是將抽象的失效率模型與具體的半導體技術聯繫起來的橋樑，是實現 ISO 26262 Part 5 中量化分析要求的關鍵步驟。
- **對稽核的重要性**: **高**。稽核員會驗證開發團隊是否為其 IC 選擇了正確的技術類別，並使用了對應的 λ₁ 和 λ₂ 值。選擇錯誤的行會直接導致失效率計算結果的巨大偏差。

## 4. 公式深度分析

此工作表包含 127 個公式，但許多是重複的模式。以下分析幾個關鍵且獨特的公式模式：

- **模式 1: `AVERAGE(...)`**
  - **公式範例**: `=AVERAGE(50,60)` (AO3), `=AVERAGE(64,57,60,54,56)` (AO4)
  - **物理意義 / 計算目的**: 這些公式用於計算一組數據的平均值。在 FMEDA 中，這通常用於處理來自不同數據手冊、或在不同操作條件下的參數，以獲得一個代表性的平均值。例如，計算多個供應商提供的類似組件的平均電晶體尺寸，或在一個任務剖面中不同階段的平均溫度。
  - **引用的其他 sheet 或儲存格**: 直接引用數值或同一工作表內的其他單元格。
  - **對應的 ISO 26262 條款**: ISO 26262-5, Annex D，要求對所有假設和輸入數據提供合理解釋。使用平均值是一種簡化複雜數據的常見方法，但必須證明其合理性。
  - **正確性驗證要點**: 稽核員會關注取樣的數據點是否具有代表性，以及為何選擇平均值而非最差情況值（Worst-case）。

- **模式 2: 複雜數學運算**
  - **公式範例**: `=AO3/AO4*AP3/AP4*AR4` (AR3)
  - **物理意義 / 計算目的**: 這是 IEC TR 62380 數學模型的一個子部分。此類公式將多個中間參數（如溫度因子、製程因子、尺寸因子）相乘或相除，以計算出一個綜合影響因子。這反映了不同物理效應之間的相互作用。
  - **引用的其他 sheet 或儲存格**: 引用同一工作表內的其他計算結果 (AO3, AO4, AP3, AP4, AR4)。
  - **對應的 ISO 26262 條款**: ISO 26262-5, Clause 8.4.2，要求失效率的估算應基於公認的數據源和方法。此公式直接執行了 IEC TR 62380 的方法。
  - **正確性驗證要點**: 驗證公式是否**精確**地複製了標準中的數學關係。任何運算符號（乘、除）或參數順序的錯誤都是不可接受的。

- **模式 3: 帶權重的科學計數法平均**
  - **公式範例**: `=AVERAGE(98.48/8500000000,122/10000000000,...) * 1000000` (AR4)
  - **物理意義 / 計算目的**: 此公式在計算平均電晶體失效率。每個數據點都是一個特定類型電晶體的總失效率除以其數量，從而得到單位電晶體的失效率。最後乘以 1,000,000 是為了將單位從「每小時」轉換為 FIT（每 10^9 小時）。
  - **引用的其他 sheet 或儲存格**: 直接嵌入數值。
  - **對應的 ISO 26262 條款**: ISO 26262-5, Annex D.2.3 提到了基於電晶體數量的失效率估算方法。
  - **正確性驗證要點**: 稽核員會檢查每個除法的分子（總失效率）和分母（電晶體數量）的數據來源是否可靠。同時，會驗證最終的單位轉換是否正確。

## 5. 稽核合規性分析

- **對應的 ISO 26262 條款**:
  - **Part 5, Clause 8**: 硬體元件的量化失效率指標評估。此工作表是計算 SPFM, LFM 和 PMHF 指標的數據輸入源頭。
  - **Part 5, Annex D**: 半導體元件失效率估算指南。此工作表直接實踐了 Annex D 推薦的 IEC TR 62380 方法。
  - **Part 9, Clause 6**: 相依失效分析。雖然此表不直接進行 DFI 分析，但其提供的基礎失效率是評估相依失效影響的基礎。
  - **Part 11, Clause 4.5**: 對半導體元件的應用。此工作表是 Part 11 指導原則在實踐中的具體應用。

- **滿足 ASIL B 等級的具體要求**: ASIL B 要求有一個可信的、系統化的方法來估算硬體失效率。使用像 IEC TR 62380 這樣的行業公認標準，並通過此 Excel 工作表進行透明、可追溯的計算，是滿足這一要求的關鍵一步。僅僅提供一個失效率數字是不夠的，必須展示其計算過程。

- **稽核員可能會問的問題**:
  1. 「你們用於計算的電晶體總數（N）是如何得出的？請提供相關的晶片設計文件證明。」
  2. 「任務剖面（Mission Profile）中的溫度和工作時間比率（(t_ae)_i, τ_i）是如何定義的？是否有來自客戶需求或實車測試的數據支持？」
  3. 「在 IEC TR 62380 Table 16 中，你們為何選擇『ASIC - Standard Cell』這一行？請解釋你們的產品技術如何與此定義匹配。」
  4. 「公式中使用的學習因子（a = 製造年份 - 1998）中的『製造年份』是如何確定的？是指樣片年份還是量產年份？」
  5. 「對於電氣過應力（EOS）的失效率（λ_EOS），你們的估算依據是什麼？是否有相關的防護設計（如 ESD）文檔作為佐證？」

- **需要準備的佐證文件**:
  - 晶片設計規格書（標明電晶體數量、晶片面積、技術節點）。
  - 產品需求文檔或客戶規格書（定義任務剖面、操作溫度範圍）。
  - 可靠性測試報告（用於校準或驗證模型中的某些參數）。
  - 供應商提供的數據手冊（Data Sheets），特別是關於封裝類型和材料的部分。
  - 一份解釋所有參數選擇理由的獨立文檔（Assumption Log）。

## 6. 稽核檢查清單

| 檢查項目 | Pass/Fail 判定標準 |
| :--- | :--- |
| **方法論合規性** | **Pass**: 明確聲明並完全遵循了 IEC TR 62380 標準進行計算。<br>**Fail**: 無法證明計算方法論的來源，或對標準進行了未經證明的修改。 |
| **參數可追溯性** | **Pass**: 所有輸入參數（N, T, τ 等）都有明確的來源文件支持。<br>**Fail**: 任何參數僅為「工程經驗」或無法提供書面依據。 |
| **查表準確性** | **Pass**: 根據產品技術，在 Table 16 和 Table 17a/b 中選擇了正確的行和值。<br>**Fail**: 選擇了不匹配的技術類別（例如，將 FPGA 的數據用於 ASIC）。 |
| **公式實現正確性** | **Pass**: Excel 公式與 IEC TR 62380 數學模型完全一致，包括所有指數、係數和運算符。<br>**Fail**: 公式存在任何偏差、簡化或錯誤。 |
| **單位一致性** | **Pass**: 所有時間單位、溫度單位都保持一致，最終結果正確轉換為 FIT。<br>**Fail**: 單位混用（例如，小時和年），或 FIT 轉換因子錯誤。 |
| **任務剖面合理性** | **Pass**: 任務剖面的定義（溫度循環、工作時間）與產品的預期用途一致且有依據。<br>**Fail**: 任務剖面過於樂觀或與實際應用場景脫節。 |
| **文件完整性** | **Pass**: 提供了所有必要的佐證文件，形成了一個完整的證據鏈。<br>**Fail**: 缺少關鍵的佐證文件，導致計算過程無法被獨立驗證。 |
