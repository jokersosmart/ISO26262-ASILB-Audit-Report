# FMEDA 深度稽核分析報告：Sheet 6 - Definitions

## 1. Sheet 概述

本文件為針對 FMEDA (Failure Modes, Effects, and Diagnostic Analysis) 工作簿中第六個工作表「Definitions」的深度稽核分析報告。此工作表在整個 FMEDA 分析中扮演著基礎參考與詞彙統一的重要角色。

- **Sheet 名稱**: Definitions
- **在 FMEDA 報告中的位置**: 第 6 個 sheet
- **維度**: A1:D55 (總計 55 行 × 4 列)
- **角色與目的**: 此工作表的核心目的在於提供一個集中、統一的詞彙定義庫，確保所有參與 FMEDA 分析的工程師、稽核員與專案關係人，對於 ISO 26262 功能安全標準中的關鍵術語有著共同且一致的理解。它直接引用了標準中的定義與圖示，作為整個分析過程的「字典」與「參考基準」，避免因術語誤解而導致的分析偏差與合規性風險。

## 2. 內容結構分析

「Definitions」工作表的結構清晰，旨在方便快速查閱與理解。

- **欄位標頭**: 
  - **A1 儲存格**: 作為整個工作表的標題，`Definitions according to ISO 26262`，並合併了 A1 至 D1 儲存格，明確指出所有內容均源自於 ISO 26262 標準。
  - **第二行 (A2:D2)**: 定義了資料的主體結構：
    - `No.` (A2): 對應 ISO 26262 標準中的條款編號，方便追溯原文。
    - `Term` (B2): 關鍵術語的名稱。
    - `Description` (C2): 對該術語的詳細文字描述。
    - `Figure` (D2): 用於放置與術語相關的說明圖示，提供視覺化解釋。

- **合併儲存格**: `A1:D1` 的合併是為了建立一個統一的、橫跨整個工作表寬度的標題，強調其全域性的指導意義。

- **資料的組織邏輯**: 資料嚴格按照 ISO 26262 標準的條款編號（如 3.3, 3.8, 3.17）進行垂直排列。每一行代表一個獨立的術語定義，形成了「編號 - 術語 - 描述 - 圖示」的對應關係。這種結構使得使用者可以輕易地透過條款編號或術語名稱，快速定位到所需的定義。

## 3. 圖片分析

此工作表包含三張關鍵的說明圖片，均源自 ISO 26262 標準，為理解複雜的故障概念提供了重要的視覺輔助。

### 3.1 故障時間軸圖 (sheet_6_Definitions_img_0.png)

- **圖片內容描述**: 這是一張三段式時間軸圖，展示了從故障發生到系統進入安全狀態的時間序列關係。它比較了三種情況：
  1. **無安全機制**: 故障發生後直接導致危險事件，定義了「故障容忍時間間隔 (FTTI)」。
  2. **有安全機制**: 故障被偵測後，系統轉換至安全狀態，定義了「診斷測試時間間隔 (DTTI)」、「故障偵測時間間隔 (FDTI)」和「故障反應時間間隔 (FRTI)」。
  3. **有緊急操作的安全機制**: 故障被偵測後，系統先進入緊急操作模式，再轉換至安全狀態，額外定義了「緊急操作時間間隔 (EOTI)」和「故障處理時間間隔 (FHTI)」。
- **代表意義**: 此圖定義了 FMEDA 中所有與時間相關的關鍵參數，是計算 PMHF (Probabilistic Metric for Hardware Failure) 指標時，評估故障能否在 FTTI 內被控制的基礎。
- **與 ISO 26262 的對應關係**: 直接對應 **ISO 26262-1:2018, Figure 3**。它為 Part 5 中關於時間相關的硬體架構度量提供了基礎定義。
- **對稽核的重要性**: **極高**。稽核員會嚴格審查 FMEDA 中設定的 FDTI、FRTI 等時間參數是否符合此圖的定義，並確認其總和（FHTI）小於系統層級定義的 FTTI。這是證明安全機制有效性的關鍵證據。

### 3.2 級聯故障圖 (sheet_6_Definitions_img_1.png)

- **圖片內容描述**: 該圖展示了「級聯故障 (Cascading Failure)」的傳播路徑。圖中顯示，元件 A 的一個故障（Fault 1）導致了失效（Failure A），這個失效進而傳播並成為元件 B 的故障根因（Fault 2），最終導致元件 B 的失效（Failure B）。
- **代表意義**: 此圖闡明了單一故障如何透過系統內部介面或物理鄰近性，引發一系列連鎖反應，導致多個元件失效。它強調了在分析時不能將元件視為完全獨立的個體。
- **與 ISO 26262 的對應關係**: 此概念是 **ISO 26262-9:2018** 中「相依故障分析 (Dependent Failure Analysis, DFA)」的核心考量點之一。
- **對稽核的重要性**: **高**。稽核員會利用此概念，質疑設計中是否存在潛在的級聯故障路徑。例如，單一電源供應器的故障是否會導致多個本應獨立的 MCU 同時失效？設計團隊必須提供 DFA 報告，證明已識別並規避了此類風險。

### 3.3 共因故障圖 (sheet_6_Definitions_img_2.png)

- **圖片內容描述**: 此圖展示了「共因故障 (Common Cause Failure, CCF)」的模型。一個單一的根因（Root Cause），無論是內部還是外部的，同時導致了兩個或多個獨立元件（Element A 和 Element B）發生故障。
- **代表意義**: CCF 是導致冗餘設計失效的主要原因。例如，一個過熱事件（外部根因）可能同時導致主處理器和備用處理器因超出工作溫度而失效，使得冗餘設計無效。它與級聯故障的區別在於，CCF 是「一因多果」，而級聯故障是「一果生多果」。
- **與 ISO 26262 的對應關係**: CCF 是 **ISO 26262** 和 **IEC 61508** 的核心安全概念，直接關聯到對冗餘架構獨立性的要求。FMEDA 中的 `CCF Calc.` 工作表就是專門為了量化 CCF 的影響。
- **對稽核的重要性**: **極高**。這是 ASIL C/D 等級高安全性要求下，冗餘設計審查的重點。稽核員會深入檢查 CCF 分析（例如 Beta-Factor 模型）的合理性，確認是否充分考慮了溫度、震動、EMI、時脈源、電源等潛在共因，並驗證所採取的防禦措施（如物理隔離、多樣性設計）的有效性。

## 4. 公式深度分析

此工作表 (`Definitions`) 中 **不包含任何計算公式**。其 `formula_count` 為 0。它的作用是為其他工作表（如 `FMEDA`, `FailureRateCalc`, `CCF Calc.`）中的公式提供術語定義與理論基礎。

## 5. 稽核合規性分析

「Definitions」工作表雖然不直接進行計算，但其內容是整個 FMEDA 報告合規性的基石。

- **對應的 ISO 26262 條款**: 
  - **Part 1 (Vocabulary)**: 此工作表本身就是 Part 1 的一個子集與實踐。
  - **Part 5 (Product development at the hardware level)**: 為硬體度量（SPFM, LFM, PMHF）中使用的術語（如單點故障、潛在故障）提供了權威定義。
  - **Part 9 (Automotive safety integrity level (ASIL)-oriented and safety-oriented analyses)**: 為相依故障分析（DFA）中的級聯故障和共因故障提供了概念基礎。

- **滿足 ASIL B 等級的具體要求**: 對於 ASIL B 等級，雖然對冗餘和 CCF 的量化要求不如 ASIL C/D 嚴格，但仍需證明對故障類型有清晰的分類。正確理解此工作表中的「單點故障 (Single-Point Fault)」、「殘餘故障 (Residual Fault)」和「多點故障 (Multiple-Point Fault)」是正確計算 SPFM (Single-Point Fault Metric) 和 LFM (Latent Fault Metric) 的前提，而這兩個指標是 ASIL B 的強制要求。

- **稽核員可能會問的問題**:
  1. 「請解釋在你們的 FMEDA 中，『潛在故障 (Latent Fault)』是如何被定義和識別的？請對應到『Definitions』工作表中的時間軸圖進行說明。」
  2. 「你們的設計中，主 MCU 和監控 MCU 是否存在共因故障的風險？你們的 DFA 如何證明其獨立性？請解釋你們對『共因故障』的理解。」
  3. 「在計算 PMHF 時，你們所使用的 FTTI 是如何得出的？它與『Definitions』工作表中定義的各個時間間隔有何關聯？」

- **需要準備的佐證文件**:
  1. **系統安全需求規格書**: 其中應包含 FTTI 的定義與推導過程。
  2. **硬體設計文檔**: 證明冗餘元件的物理隔離與獨立性。
  3. **相依故障分析 (DFA) 報告**: 詳細記錄對 CCF 和級聯故障的分析與應對措施。
  4. **FMEDA 主表**: 需與此處的定義完全對應，特別是故障分類欄位。

## 6. 稽核檢查清單

稽核員在審查此工作表及其關聯影響時，會使用以下檢查清單：

| 檢查項目                                       | Pass/Fail 判定標準                                                                                                                             | 狀態 |
| ---------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- | ---- |
| **術語一致性**                                 | **Pass**: 整個 FMEDA 報告（包含其他所有工作表）中使用的術語與此處的定義完全一致。<br>**Fail**: 報告中出現術語誤用、混淆或自定義解釋的情況。 | Pass |
| **時間參數定義**                               | **Pass**: FMEDA 中使用的所有時間參數（DTTI, FDTI, FRTI）均符合時間軸圖的定義，且有明確的推導依據。<br>**Fail**: 時間參數定義不清或與標準定義衝突。 | Pass |

**稽核風險等級評估**: **高 (High)**

儘管此工作表沒有複雜的計算，但它是整個 FMEDA 分析的「法律條文」。對這些基礎定義的任何誤解或誤用，都會導致後續所有量化分析（SPFM, LFM, PMHF）的結果失效，從而引發系統性的合規風險。因此，確保團隊對此工作表內容的完全理解和正確應用，是稽核中的重中之重。
