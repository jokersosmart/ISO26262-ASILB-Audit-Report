# FMEDA 深度稽核分析報告：FailureRateCalcIC

---

## 1. Sheet 概述

本文件為針對 FMEDA (Failure Modes, Effects, and Diagnostic Analysis) 工作簿中 **第 14 個工作表** `FailureRateCalcIC` 的深度稽核分析報告。

- **Sheet 名稱**: `FailureRateCalcIC`
- **在 FMEDA 報告中的位置**: 第 14 個 sheet
- **維度**: 142 行 × 14 列 (A2:N142)

此工作表在整個 FMEDA 分析流程中扮演著**基礎失效率計算引擎**的關鍵角色。其核心目的在於根據國際標準 **IEC TR 62380**（一個廣泛用於電子元件可靠度預測的技術報告）中定義的數學模型，為專案中使用的積體電路（Integrated Circuits, ICs）提供一個量化、可追溯且標準化的基礎失效率（Base Failure Rate, λ）。這個計算出的失效率是後續所有安全分析（如 SPFM, LFM, PMHF 等指標計算）的起點與基石，其準確性直接影響整個功能安全分析的有效性。

## 2. 內容結構分析

`FailureRateCalcIC` 的結構是為了引導使用者輸入 IEC TR 62380 模型所需的所有參數而設計的，其組織邏輯清晰，旨在將複雜的物理模型拆解為一系列可填寫的表格欄位。

- **欄位標頭與組織方式**: 
  - 儲存格 `B2` 中的標題 `Failure rate calculation for Integrated Circuits acc. to IEC TR 62380 formulas. IEC TR 62380 Chapter 7` 明確揭示了此表的功用與標準依據。
  - 表格主要分為幾個區塊：
    1.  **輸入參數區 (Input Parameters)**: 通常位於表格上半部，用於定義環境條件（溫度、任務剖面）、元件特性（技術、製造年份、電晶體數量）等。
    2.  **中間計算區 (Intermediate Calculations)**: 將輸入參數代入公式的各個部分，計算如溫度因子 `π_t`、學習因子 `e^(-0.35×a)` 等中間值。
    3.  **結果輸出區 (Result Output)**: 彙總所有中間計算，最終得出 IC 的總失效率 λ。

- **合併儲存格的用途**: 
  - 大量合併儲存格（如 `B2:F2`, `F106:I106`, `A88:D88`）主要用於建立清晰的區塊標題和分類，將相關的參數或計算組合在一起，提升表格的可讀性。例如，`F88:H88` 和 `J88:L88` 可能分別代表了兩種不同條件下的失效率計算結果，透過合併儲存格進行了視覺上的區分。

- **資料的組織邏輯**: 
  - 資料以「查表」和「計算」相結合的方式組織。使用者需要根據 `image_analysis_context` 中提供的 IEC TR 62380 表格（如 Table 16, 17a/17b）查詢特定技術的基礎失效率（λ₁、λ₂、λ₃），並將這些值填入對應的儲存格。接著，表格內的公式會自動引用這些輸入值，並結合環境參數，完成複雜的失效率計算。這種設計將標準的應用與實際的計算無縫整合，減少了手動計算的錯誤。

## 3. 圖片分析

此工作表 (`FailureRateCalcIC`) 本身並未直接嵌入圖片。然而，`image_analysis_context` 中與失效率計算最相關的圖片位於 `Sheet 13: FailureRateCalc`，它們是理解本工作表所有公式的必要參考與依據。這些圖片共同構成了 IEC TR 62380 標準的核心內容。

1.  **圖片**: `sheet_13_FailureRateCalc_img_2.png`
    - **圖片內容描述**: 該圖片展示了 IEC TR 62380 中用於計算 IC 失效率的完整數學模型。
    - **代表意義**: 這是本工作表所有公式的理論基礎。公式將 IC 的總失效率 λ 分解為三個主要部分：
        1.  **λ_die**: 晶粒本身因內部缺陷和老化導致的失效率。
        2.  **λ_package**: 封裝因熱應力、熱循環導致的失效率。
        3.  **λ_overstress**: 因電氣過應力（EOS）導致的失效率。
    - **與 ISO 26262 / IEC 61508 的對應關係**: ISO 26262-5 附錄 D 提供了多種估算硬體元件失效率的方法，其中就包括使用 IEC 61508-2 附錄 C 中引用的行業手冊，而 IEC TR 62380 正是此類手冊的代表。使用此模型是滿足 ISO 26262 對於失效率估算需「基於已建立的數據來源」這一要求的具體實踐。
    - **對稽核的重要性**: **極高**。稽核員會將此圖片中的數學模型與 `FailureRateCalcIC` 工作表中的公式逐一比對，以驗證公式是否**完整且正確地**實現了標準模型。任何偏差都可能導致整個 FMEDA 的量化結果失效。

2.  **圖片**: `sheet_13_FailureRateCalc_img_0.png`
    - **圖片內容描述**: 該圖片列出了實現上述數學模型所需的「必要資訊」（NECESSARY INFORMATION）表格。
    - **代表意義**: 它定義了模型的每一個輸入參數，例如環境溫度 `(t_ae)_i`、電晶體數量 `N`、製造年份 `a`、任務剖面 `τ_i` 等。
    - **與 ISO 26262 / IEC 61508 的對應關係**: 這張表確保了失效率計算的可追溯性和可重複性。ISO 26262 要求所有安全相關的假設和數據都有其依據。
    - **對稽核的重要性**: **極高**。稽核員會使用此表來檢查 FMEDA 報告是否為每一個輸入參數都提供了合理的數值和明確的來源依據（例如，溫度剖面是否來自任務剖面分析報告）。

3.  **圖片**: `sheet_13_FailureRateCalc_img_1.png`
    - **圖片內容描述**: 該圖片是 IEC TR 62380 的 Table 16，提供了不同類型 IC（如 ASIC, FPGA, Memory）的基礎失效率參數 λ₁ 和 λ₂。
    - **代表意義**: 這是模型中 `λ_die` 計算的關鍵查表數據。不同的半導體技術和設計方法（如 Standard Cell vs. Gate Array）有不同的基礎失效率。
    - **與 ISO 26262 / IEC 61508 的對應關係**: 確保了針對特定元件類型選用合適的行業統計數據。
    - **對稽核的重要性**: **高**。稽核員會驗證專案中所用的 IC（例如，一顆 SSD 控制器 ASIC）是否在此表中選擇了正確對應的條目（例如，應選用 `Silicon MOS Standard circuits` -> `ASIC circuits` -> `Standard Cell`）。錯誤的選擇將導致基礎失效率的數量級偏差。

## 4. 公式深度分析

此工作表包含 111 個公式，展現了 33 種獨特的計算模式。這些公式共同構建了 IEC TR 62380 的失效率模型。以下列舉幾個關鍵公式模式進行深度分析：

- **公式模式 1: 學習因子相關計算**
  - **範例**: `D93 = C93 - 1998`
  - **物理意義**: 此公式計算參數 `a`，即「製造年份 - 1998」。這是 IEC TR 62380 模型中的「學習因子」，反映了半導體製程隨時間演進會變得更加成熟可靠的趨勢。`a` 值越大，代表技術越新，基礎失效率會相應降低（體現在 `e^(-0.35×a)` 這一項）。
  - **引用的儲存格**: `C93`，即 IC 的製造年份。
  - **對應的標準**: IEC TR 62380 Chapter 7 的核心數學模型。
  - **正確性驗證要點**: 稽核員會檢查 `1998` 這個常數是否正確（與標準一致），以及 `C93` 的輸入值是否有文件支持（例如，來自晶圓廠的資訊）。

- **公式模式 2: 基礎失效率計算 (Die Failure Rate)**
  - **範例**: `G94 = 0.4*(20+2500/(D99+3))` (此為簡化示意，實際公式更複雜)
  - **物理意義**: 這是對 `λ_die` 的部分計算。它結合了查表得到的基礎失效率（如 `0.4`）、與電晶體數量相關的項、以及與溫度和任務剖面相關的因子（如 `2500/(D99+3)`，其中 `D99` 可能代表溫度）。
  - **引用的儲存格**: `D99` (環境或結點溫度), 以及其他包含電晶體數量、λ₁、λ₂ 等參數的儲存格。
  - **對應的標準**: IEC TR 62380 的 `λ_die` 部分。
  - **正確性驗證要點**: 稽核員會將此公式與標準模型進行代數比對，確保所有係數（如 `0.4`, `20`, `2500`, `3`）都與標準或其引用的表格完全一致。同時，會驗證其引用的溫度 `D99` 是否來自有效的熱分析報告。

- **公式模式 3: 引用其他工作表的計算結果**
  - **範例**: `B94 = FailureRateCalc!B21`
  - **物理意義**: 此公式從另一個名為 `FailureRateCalc` 的工作表中引用元件的描述。
  - **引用的其他 sheet 或儲存格**: `FailureRateCalc!B21`
  - **對應的標準**: 無直接對應，但體現了工作簿內部資料聯動的設計，確保了一致性。
  - **正確性驗證要點**: 稽核員會檢查被引用的工作表 `FailureRateCalc` 是否為本 FMEDA 的主元件清單，確保資料來源的唯一性和正確性。

- **公式模式 4: 條件判斷公式**
  - **範例**: `D91 = IF(C91=J106,3,IF(C91=I107,2,IF(C91=H107,1,0)))`
  - **物理意義**: 根據 `C91` 儲存格中的輸入值（可能是一個代表環境或任務剖面類型的代碼），從 `J106`, `I107`, `H107` 等儲存格中選擇一個對應的因子（`3`, `2`, `1`）。這通常用於實現查表功能，例如根據「任務剖面」的名稱（如 "Ground, Stationary"）來選擇對應的溫度因子或振動因子。
  - **引用的儲存格**: `C91`, `J106`, `I107`, `H107`。
  - **對應的標準**: IEC TR 62380 中各種需要查表獲得的 `π` 因子（環境因子）。
  - **正確性驗證要點**: 稽核員會驗證 `IF` 判斷的條件和返回的值是否與標準表格的內容完全匹配，確保查表邏輯的正確性。

## 5. 稽核合規性分析

此工作表雖然只是一個計算工具，但它在 ISO 26262 合規性中扮演著基礎性的支撐角色。

- **對應的 ISO 26262 條款**:
  - **Part 5, Clause 8 (Hardware architectural metrics)**: 本工作表計算出的基礎失效率是計算 SPFM 和 LFM 指標的起點。沒有準確的 λ，就無法進行後續的指標計算。
  - **Part 5, Annex D (Estimation of failure rates of hardware parts)**: 本工作表直接實踐了該附錄中推薦的「使用公認的行業數據手冊（如 IEC TR 62380）來估算失效率」的方法。
  - **Part 9, Clause 6 (Analysis of dependent failures)**: 雖然本表不直接分析相依失效，但其計算出的封裝失效率 `λ_package` 與溫度循環、熱膨脹係數等參數相關，這些是共因失效（CCF）的物理根源之一。
  - **Part 11, Clause 4.3 (Application of ISO 26262 to semiconductors)**: 該章節詳細說明了如何將 ISO 26262 應用於 IP、SoC、FPGA 等半導體元件。本工作表正是針對 IC 元件進行失效率量化的標準實踐。

- **滿足 ASIL B 等級的具體要求**:
  - 對於 ASIL B，ISO 26262 要求對硬體架構指標（SPFM, LFM）進行評估。本工作表提供了計算這些指標所需的基礎數據。雖然標準對 ASIL B 的 SPFM/LFM 僅為「建議」而非強制，但提供一個基於行業標準的 FMEDA 量化分析是展現設計穩健性的最佳實踐。
  - 更重要的是，PMHF（Probabilistic Metric for random Hardware Failures）的計算是強制的。本工作表計算的 λ 是 PMHF 計算公式中不可或缺的輸入項。

- **稽核員可能會問的問題**:
  1.  「你們是如何確定這顆 IC 的電晶體數量（Gate Count）的？請提供相關文件。」
  2.  「任務剖面（Mission Profile）中的溫度和工作時間比例是如何得出的？請展示你們的系統級熱分析報告和任務剖面定義文件。」
  3.  「為什麼你們為這顆 ASIC 選擇了 IEC TR 62380 Table 16 中的這一行數據？請解釋你們的判斷依據。」
  4.  「請解釋 `D93 = C93 - 1998` 這個公式的物理意義。為什麼是減去 1998？」
  5.  「此工作表中的公式與 IEC TR 62380 標準數學模型之間的一致性是如何驗證的？請提供你們的驗證紀錄。」

- **需要準備的佐證文件**:
  1.  **元件數據手冊 (Datasheet)**: 提供 IC 的基本資訊。
  2.  **晶圓廠/設計公司提供的報告**: 包含電晶體數量、技術節點、製造年份等資訊。
  3.  **系統任務剖面分析報告 (Mission Profile Analysis Report)**: 定義系統在生命週期內經歷的各種環境條件（溫度、濕度、振動）及其時間佔比。
  4.  **熱分析報告 (Thermal Analysis Report)**: 提供 IC 在不同工作負載下的結點溫度（Junction Temperature）。
  5.  **FMEDA 方法論文件**: 說明為何選擇 IEC TR 62380 作為失效率預測的依據。

| 1 | **標準一致性** | **Pass**: 工作表中的所有公式、常數和查表邏輯均與 IEC TR 62380 的數學模型和表格完全一致。<br>**Fail**: 存在任何無法解釋的偏差。 | **高** |
| 2 | **輸入參數可追溯性** | **Pass**: 所有輸入參數（溫度、Gate Count、製造年份、任務剖面等）都有對應的佐證文件支持。<br>**Fail**: 任何參數為「假設值」或「經驗值」且無文件依據。 | **高** |

**綜合稽核風險評估**: **高**

此工作表是整個量化 FMEDA 的地基。任何在此處的錯誤、無依據的假設或對標準的誤解，都將導致後續所有安全指標（SPFM, LFM, PMHF）的計算結果失效，從而使整個安全分析失去意義。因此，稽核員會對此表進行最為嚴格和詳細的審查。
