# FMEDA 深度稽核分析報告：Sheet 8 - FlowChart

## 1. Sheet 概述

- **Sheet 名稱**: `FlowChart`
- **在 FMEDA 報告中的位置**: 第 8 個 sheet
- **維度**: 189 行 × 18 列 (B2:R189)
- **角色與目的**: 此工作表在整個 FMEDA 分析中扮演著**方法論核心與視覺化參考**的關鍵角色。它本身不包含計算資料或硬體元件清單，而是透過四張核心的流程圖，完整定義了本 FMEDA 報告所遵循的分析流程、計算邏輯與分類標準。對於稽核員而言，此 sheet 是理解和驗證整個 FMEDA 分析方法論正確性的起點與核心依據，其重要性等同於軟體工程中的架構設計圖。

## 2. 內容結構分析

此工作表的結構極為簡單，其設計完全服務於展示核心流程圖的目的。

- **欄位標頭**: 僅在儲存格 `B2` 存在一個標題：「Flow chart acc. to ISO 26262-5:2018」，並以深藍色背景和粗體字標示，明確指出此頁內容是依據 ISO 26262 第五部分（Part 5）的標準所繪製。除此之外，沒有傳統意義上的資料表頭。
- **合併儲存格**: 此 sheet 未使用任何合併儲存格。
- **資料組織邏輯**: 內容並非以表格資料形式組織，而是以四張圖片為主體，垂直排列。這種佈局旨在提供一個清晰、無干擾的閱讀體驗，讓使用者可以專注於理解流程圖本身所傳達的複雜邏輯。

## 3. 圖片分析

此 sheet 包含四張圖片，它們共同構成了 FMEDA 分析的完整方法論框架。以下根據 `image_analysis_context` 的辨識結果進行詳細說明。

### 圖片 1: FMEDA 詳細計算流程圖 (sheet_8_FlowChart_img_0.png)

- **圖片內容描述**: 這是一張極其詳細的流程圖，展示了 SGS FMEDA 模板的核心計算邏輯。它從單一的「失效模式 (Failure mode)」出發，經過一系列的決策判斷（菱形方塊）與計算過程（方形方塊），最終將該失效模式的失效率（λ）分配到七個不同的故障類別中。
- **代表意義**: 此圖是整個 FMEDA 工作簿中所有計算公式的「視覺化演算法」。它定義了從基礎失效率到單點故障（SPF）、殘餘故障（RF）和多點故障（MPF）的完整計算路徑，是方法論的具體實作。
- **與 ISO 26262 / IEC 61508 的對應關係**: 這是 ISO 26262-5:2018 中 Figure 5 概念流程圖的一個高度詳細和可操作的實作版本。它將標準中的抽象概念轉化為具體的計算步驟。
- **對稽核的重要性**: **極高**。這是稽核員驗證 FMEDA 主計算表中所有公式正確性的「聖經」。稽核員會逐一比對主表中的公式與此流程圖的邏輯是否完全一致，任何偏差都可能構成嚴重的不符合項。例如，稽核員會檢查「殘餘故障率 (λ_FM,RF)」的計算公式是否確實等於 `(1 - K_FMCl,RF) × λ_FM,PVSG`。

### 圖片 2: 故障分類樹狀圖 (sheet_8_FlowChart_img_1.png)

- **圖片內容描述**: 這是一張層次結構圖，展示了硬體元件的失效模式如何被分類。頂層是「Failure mode of a HW element」，首先分為「安全相關」與「非安全相關」，其中「安全相關」的故障進一步細分為六大類別，包括安全故障、已偵測多點故障、已感知多點故障、潛伏多點故障、殘餘故障和單點故障。
- **代表意義**: 此圖提供了 ISO 26262 故障分類體系的概覽，是計算硬體架構度量指標（SPFM 和 LFM）的理論基礎。
- **與 ISO 26262 / IEC 61508 的對應關係**: 此圖的分類法直接對應於 ISO 26262-5:2018 中的 Table 3 — 「Classification of hardware architectural faults」。
- **對稽核的重要性**: **高**。此圖是稽核員確認開發團隊是否正確理解並應用了標準所定義的故障分類法的直接證據。SPFM 和 LFM 的計算完全基於此分類，任何誤解都會導致度量指標失效。

### 圖片 3: ISO 26262-5 Process 流程圖 (sheet_8_FlowChart_img_2.png)

- **圖片內容描述**: 這張流程圖與 Sheet 3 (Process) 中的流程圖內容相同，是 ISO 26262-5:2018 標準原文中 Figure 5 的直接翻版。它從「HW element λ」開始，通過一系列判斷（如「is it safety-related?」、「potential to violate the SG?」）將故障分類。
- **代表意義**: 它提供了官方標準的原始流程圖作為參照，用於與本模板的實作流程圖（圖片 1）進行比對，以證明模板的合規性。
- **與 ISO 26262 / IEC 61508 的對應關係**: 直接來自 ISO 26262-5:2018, Figure 5。
- **對稽核的重要性**: **中等**。它的主要作用是作為一個「標準參照」，證明本 FMEDA 模板的設計者已經理解並試圖遵循標準的指導。稽核的重點仍在於圖片 1 的實作是否準確。

### 圖片 4: ISO 26262-5 Notes (sheet_8_FlowChart_img_3.png)

- **圖片內容描述**: 這張圖截取了 ISO 26262-5 中關於故障分類的兩條重要註釋（NOTE 1, NOTE 2）和一個範例（EXAMPLE）。
- **代表意義**: 這些註釋定義了 FMEDA 分析中的一些關鍵邊界條件和例外規則。例如，NOTE 1 指出，階數大於 2 的多點故障（即需要三個或更多獨立故障才會違反安全目標的故障）可以被視為安全故障。NOTE 2 強調同一個故障在不同安全目標下可能有不同的分類。
- **與 ISO 26262 / IEC 61508 的對應關係**: 直接來自 ISO 26262-5:2018 標準正文。
- **對稽核的重要性**: **高**。這些註釋是進行「安全故障」判定的重要依據，直接影響 SPFM 和 LFM 的計算結果。稽核員會用這些規則來挑戰 FMEDA 中對某些特定故障的分類是否合理。例如，如果一個故障被歸類為安全故障，稽核員可能會問：「請根據 NOTE 1 解釋為什麼這個故障是安全的，並提供證據證明它是 n>2 的多點故障。」

## 5. 稽核合規性分析

- **對應的 ISO 26262 條款**:
  - **Part 5, Clause 8 (Hardware architectural metrics)**: 此 sheet 的流程圖是計算 SPFM 和 LFM 的方法論基礎。
  - **Part 5, Annex D (Classification of failures)**: 流程圖詳細闡述了如何執行 Annex D 中描述的故障分類過程。
  - **Part 9, Clause 6 (Analysis of dependent failures)**: 雖然未直接顯示，但流程圖中對多點故障的處理隱含了對相依故障的考慮。

- **滿足 ASIL B 等級的具體要求**: ASIL B 要求 SPFM ≥ 90% 且 LFM ≥ 60%。此 sheet 的流程圖和分類法是確保這些指標被正確計算和評估的先決條件。一個經過驗證的、合規的流程是達到任何 ASIL 等級的基礎。

- **稽核員可能會問的問題**:
  1. 「請解釋你們的 FMEDA 計算流程。你們如何確保主計算表中的公式與 `FlowChart` sheet 中定義的流程完全一致？」
  2. 「在流程圖（圖片1）的決策點 `k` (判斷 F_FM,PVSG)，你們是如何評估一個失效模式違反安全目標的潛力分數的？請提供評估準則和範例。」
  3. 「根據流程圖（圖片4）的 NOTE 1，你們在分析中是否將任何 n>2 的多點故障歸類為安全故障？如果是，請指出具體案例並提供證據。」
  4. 「流程圖（圖片2）中的故障分類與 ISO 26262-5 Table 3 完全對應。請展示你們是如何在 FMEDA 中標記每個故障，以確保它能被正確地歸入這幾個類別之一。」

- **需要準備的佐證文件**:
  - **FMEDA 方法論文件**: 一份詳細描述如何使用此 Excel 模板進行 FMEDA 分析的內部指引文件。
  - **工具驗證報告**: 如果此 Excel 模板被視為一個「工具」，可能需要一份工具驗證報告（Tool Confidence Level, TCL），證明其計算的準確性。此 `FlowChart` sheet 將是該報告的核心內容。
  - **FMEDA 審查記錄**: 證明團隊內部已經對 FMEDA 的結果和所使用的方法（即此流程圖）進行了同行審查（Peer Review）。
