# FMEDA 深度稽核分析報告：Sheet 10 - BlockList

## 1. Sheet 概述

本文件為針對 FMEDA (Failure Modes, Effects, and Diagnostic Analysis) 工作簿中第 10 個工作表「BlockList」的深度稽核分析報告。此報告旨在詳細解析該工作表的結構、目的、計算邏輯及其在 ISO 26262 功能安全合規性中的角色。

| 屬性 | 內容 |
| :--- | :--- |
| **Sheet 名稱** | `BlockList` |
| **在 FMEDA 報告中的位置** | 第 10 個 sheet |
| **維度** | 209 行 × 146 列 (A1:EP209) |

此工作表在整個 FMEDA 分析流程中扮演著**基礎資料庫與參數配置中心**的核心角色。其主要目的在於定義半導體晶片中所有硬體「區塊 (Block)」的類型，並為每種類型配置其基礎失效率 (Base Failure Rate, BFR)。它不僅列出了不同硬體電路（如數位電路、SRAM、ROM）的分類，還作為一個中央參數表，匯總了如電晶體數量、晶片面積等物理屬性，並計算其在永久性 (Permanent) 和暫態性 (Transient) 故障下的失效率貢獻。這些經過計算的基礎失效率是整個 FMEDA 中後續詳細故障模式分析的起點，為其他工作表（如主 FMEDA 分析表）提供了最根本的失效率數據輸入。

## 2. 內容結構分析

「BlockList」工作表的結構設計旨在將參數配置、數據匯總和詳細列表清晰地分開，其組織邏輯如下：

- **配置與摘要區 (第 1-8 行)**: 此區域為整個工作表的控制面板。它包含了用於計算的全局性假設和選項，例如製造年份 (T2)、半導體技術節點 (V2)、是否考慮封裝失效率 (AJ2) 等。這些儲存格通常包含下拉式選單，供分析師設定分析的上下文。同時，此區域也展示了對下方詳細數據的匯總計算結果，如 `D2` 儲存格的 `Sum(all Block Failure Rates)Permanent`。

- **詳細區塊列表 (第 10 行及以下)**: 這是實際的硬體區塊清單，理論上每一行都代表晶片中的一個具體硬體單元。分析師會在此處填寫每個區塊的具體資訊，例如其所屬的「Block Type」(R欄)，以及其電晶體數量 (L欄) 或面積 (M欄)。

#### 欄位標頭的含義與組織方式

標頭被精心組織，以顏色區分功能區塊，並透過換行來清晰表達複雜概念。關鍵標頭包括：

| 儲存格 | 標頭名稱 | 物理意義與用途 |
| :--- | :--- | :--- |
| `B1` | `Block Type` | 定義硬體區塊的類型，例如「Si MOS: Digital circuits」。這是 `SUMIF` 等公式進行分類匯總的依據。 |
| `C1` | `Base Failure Rate Permanent` | 該類型區塊的基礎永久性失效率 (單位: FIT)。此數值通常引用自 `FailureRateCalc` 工作表。 |
| `D1` | `Sum (all Block Failure Rates) Permanent` | 匯總下方詳細列表中所有區塊的總永久性失效率。 |
| `E1` | `Sum (Amount of Transistors per Block Type)` | 根據區塊類型，匯總其總電晶體數量。 |
| `J1` | `Base Failure Rate Permanent to Transient Multiplier` | 一個乘數，用於從永久性失效率推算出暫態性失效率。 |
| `L1` | `Base Failure Rate Transient` | 該類型區塊的基礎暫態性失效率 (單位: FIT)。 |
| `R1` | `Failure Rate Permanent Distribution based on` | 一個下拉式選單，決定失效率是基於「電晶體數量」還是「面積」進行分配。 |
| `AJ1` | `Consider Package and EOS in BFR calculation` | 一個「是/否」的開關，決定是否在基礎失效率計算中計入封裝 (Package) 和電氣過應力 (EOS) 的影響。 |

#### 合併儲存格的用途

合併儲存格在此工作表中主要用於改善可讀性和版面配置。例如：
- **標頭合併**: `AJ1:AL1` 將一個長的標頭「Consider Package and EOS...」合併到多個列中，使其在單一邏輯單元格內完整顯示。
- **數據顯示合併**: `D2:D7` 合併了多個儲存格，用於顯示一個適用於整個表格的總計值 `Sum(all Block Failure Rates)Permanent`，避免了數據的重複顯示，使版面更加整潔。

## 4. 公式深度分析

此工作表包含大量公式 (共 1448 個，具有 24 種獨特的模式)，是整個 FMEDA 計算的引擎之一。這些公式主要執行兩類任務：**參數引用**與**條件加總**。

#### 獨特的公式模式及其解析

以下列出幾個關鍵的公式模式及其詳細分析：

1.  **基礎失效率計算 (儲存格 C2)**
    - **公式**: `=IF($AJ$2="No",FailureRateCalc!E23,(FailureRateCalc!E23+FailureRateCalcIC!$D$129*(BlockList!E2/SUM(BlockList!$E$2:$E$7))+FailureRateCalcIC!$D$130*(BlockList!E2/SUM(BlockList!$E$2:$E$7))))`
    - **物理意義**: 此公式計算「Si MOS: Digital circuits」這一塊類型的基礎永久性失效率。它實現了一個條件邏輯：
        - 如果 `AJ2` 為 "No" (不考慮封裝和EOS)，則直接引用 `FailureRateCalc` 工作表中的基礎晶粒失效率 `E23`。
        - 如果 `AJ2` 為 "Yes"，則在基礎晶粒失效率之上，按比例增加封裝 (`FailureRateCalcIC!$D$129`) 和 EOS (`FailureRateCalcIC!$D$130`) 的失效率貢獻。分配比例基於此塊類型電晶體數 (`BlockList!E2`) 佔總電晶體數 (`SUM(BlockList!$E$2:$E$7)`) 的百分比。
    - **引用**: `FailureRateCalc!E23`, `FailureRateCalcIC!$D$129`, `FailureRateCalcIC!$D$130`, `BlockList!E2`。
    - **對應標準**: 此計算方法論與 **IEC TR 62380** 的模型一致，該模型將總失效率分解為晶粒、封裝和過應力等部分。
    - **正確性驗證**: 稽核員需驗證：(1) `AJ2` 的開關邏輯是否正確實現。(2) 引用自 `FailureRateCalc` 和 `FailureRateCalcIC` 的失效率值是否準確。(3) 電晶體數量的比例計算是否正確。

2.  **分類匯總電晶體/面積 (儲存格 E2, F2)**
    - **公式**: `=SUMIF($R$10:$R$209,$B2,$L$10:$L$209)`
    - **物理意義**: 此公式匯總了下方詳細列表中所有類型為 `B2` ("Si MOS: Digital circuits") 的硬體區塊的總電晶體數量。它從詳細列表的 `R` 欄中查找匹配 `B2` 的項目，並對應加總 `L` 欄（電晶體數量）的值。
    - **引用**: `$R$10:$R$209` (區塊類型列表), `$B2` (當前塊類型), `$L$10:$L$209` (電晶體數量列表)。
    - **對應標準**: 這是數據處理的標準實踐，旨在將詳細數據聚合到摘要級別，符合 **ISO 26262-5, Annex D** 中對基礎失效率估算的要求，即需要基於元件的物理特性（如複雜度或電晶體數）進行估算。
    - **正確性驗證**: 需檢查 `SUMIF` 的範圍是否涵蓋了所有詳細數據行，以及條件 (`$B2`) 和加總範圍 (`$L$10:$L$209`) 是否正確對應。

3.  **暫態失效率計算 (儲存格 L2)**
    - **公式**: `=IF($AJ$2="No",C2*J2,FailureRateCalc!E23*J2+FailureRateCalcIC!$D$129*(E2/SUM($E$2:$E$7))*$AN$7+FailureRateCalcIC!$D$130*(E2/SUM($E$2:$E$7))*$AN$7)`
    - **物理意義**: 計算暫態失效率。其結構與永久性失效率公式類似，但在每個部分都乘以一個「永久性到暫態性」的轉換乘數 (`J2` 或 `$AN$7`)。
    - **引用**: `C2` (永久性BFR), `J2` (暫態乘數), `FailureRateCalc!E23`, `FailureRateCalcIC!$D$129`, `$AN$7` (另一個暫態乘數)。
    - **對應標準**: **ISO 26262** 承認暫態故障（或稱軟錯誤, Soft Errors）是半導體一個重要的失效物理機制，尤其是在高密度和低電壓技術節點中。FMEDA 必須考慮這類故障。
    - **正確性驗證**: 稽核員會重點審查 `J2` 和 `$AN$7` 這兩個暫態乘數的來源和合理性。其數值應有來自可靠數據（如學術研究、行業標準或實測數據）的支持。

## 5. 稽核合規性分析

「BlockList」工作表雖然不直接進行故障模式的分類，但其提供的基礎失效率是整個 FMEDA 的基石，因此與 ISO 26262 的多個部分緊密相關。

- **對應的 ISO 26262 條款**:
    - **ISO 26262-5:2018, Clause 7 (Hardware architectural metrics)**: 此工作表計算的總失效率 `P2` (`=D2+M2`) 是計算 SPFM (Single-Point Fault Metric) 和 LFM (Latent Fault Metric) 的總失效率分母 (Σλ)。
    - **ISO 26262-5:2018, Annex D (Estimation of failure rates)**: 提供了估算硬體元件失效率的指導，此工作表正是該指導的具體實踐。它依賴於行業標準（如 IEC TR 62380）或公司歷史數據來確定基礎失效率。
    - **ISO 26262-11:2018, Clause 5 (Application of ISO 26262 to semiconductors)**: 該部分特別強調了對半導體特定失效物理（如暫態故障）的考慮，此工作表中的暫態失效率計算直接響應了這一要求。

- **滿足 ASIL B 等級的具體要求**:
    對於 ASIL B，ISO 26262 要求對失效率的估算應基於可信的數據來源。雖然不強制要求使用特定的標準，但使用如 IEC TR 62380 或類似的公認模型，並在此工作表中清晰地記錄計算過程和參數，是證明其分析嚴謹性的有力證據。此工作表的結構化方法有助於滿足 ASIL B 對於分析過程系統性的要求。

- **稽核員可能會問的問題**:
    1.  「單元格 `C2` 中的基礎失效率是如何得出的？請提供其引用的 `FailureRateCalc` 工作表中的計算依據。」
    2.  「單元格 `J2` 的『永久性到暫態性乘數』的數值依據是什麼？是否有行業數據或內部測試報告支持這個值？」
    3.  「當 `AJ2` 設置為 "Yes" 時，封裝和 EOS 的失效率是如何分配到不同塊類型的？為什麼選擇基於電晶體數量進行分配？」
    4.  「請解釋 `R1` 下拉選單中『Transistors』和『Area』兩個選項的區別，以及在何種情況下應選擇哪一個？」

- **需要準備的佐證文件**:
    1.  用於推導基礎失效率的原始數據報告（例如，來自 IEC TR 62380 的參數表、供應商提供的失效率數據、或內部可靠性測試報告）。
    2.  關於暫態故障率乘數的技術備忘錄或研究論文，解釋其數值的合理性。
    3.  一份文件，說明為何選擇基於電晶體數量（或其他指標）來分配封裝等外部失效率，並論證其合理性。

以下是稽核員在審查「BlockList」工作表時可能使用的檢查清單：

| # | 檢查項目 | Pass/Fail 判定標準 |
|:-:| :--- | :--- |
| 1 | **數據來源可追溯性** | **Pass**: 所有基礎失效率和關鍵參數（如暫態乘數）都有清晰的引用來源，且引用的工作表或外部文件可供查閱。<br>**Fail**: 參數為無來源的「魔法數字」。 |
| 2 | **公式邏輯正確性** | **Pass**: 所有 `IF`、`SUMIF` 等公式的邏輯均與其物理意義描述相符，條件和範圍設置正確。<br>**Fail**: 公式存在邏輯錯誤，例如條件判斷錯誤或加總範圍不完整。 |

| 6 | **錯誤值處理** | **Pass**: 工作表中沒有 `#VALUE!`、`#DIV/0!` 等錯誤值。所有計算均能得出有效數值。<br>**Fail**: 存在錯誤值，表明輸入數據不完整或公式存在問題（如 `J3` 中的 `#VALUE!`）。 |

**稽核風險評估**: 中等 (Medium)。此工作表雖然計算邏輯相對直接，但其作為整個 FMEDA 的數據源頭，任何參數的錯誤或不合理的假設都將被放大並傳播到後續所有分析中。特別是暫態乘數等關鍵參數的合理性，是稽核的重點和潛在風險點。存在 `#VALUE!` 和 `#DIV/0!` 錯誤也表明當前數據不完整，增加了風險。
