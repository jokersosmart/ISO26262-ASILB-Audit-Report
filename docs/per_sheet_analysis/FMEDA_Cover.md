# FMEDA Cover Sheet - 公式分析報告

這份文件詳細分析了 `FMEDA_Cover` 工作表中包含的每個獨特公式。

---

### 公式 1：取得分析日期

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | D1 |
| 公式原文 | `=IF(D3="--","--",VLOOKUP(D3,D33:G48,4,0))` |
| 應用範圍 | D1 |
| 計算值 | -- |

**參照來源**：

*   `D3`：目前選擇的分析版本號。
*   `D33:G48`：版本歷史紀錄表，其中包含了版本號與對應的分析日期。

**計算邏輯（白話文）**：

這個公式首先檢查儲存格 `D3` 的值。如果 `D3` 的內容是 `"--"`（代表沒有選擇任何版本），則這個儲存格也顯示 `"--"`。否則，它會在 `D33:G48` 這個範圍內查找與 `D3` 中版本號完全相符的紀錄，並回傳該紀錄右邊第 4 欄的內容，也就是對應的「分析日期」。

**物理意義**：

在功能安全分析中，此公式的物理意義是確保 FMEDA 報告的封面頁能夠根據所選的分析版本，自動且準確地顯示對應的分析執行日期。這對於維持文件版本的一致性與可追溯性至關重要。

**標準依據**：

這個公式本身是 Excel 的標準功能，並非直接源自於特定的安全標準條文。然而，它實現了 **ISO 26262-2:2018** 中對於「組態管理」（Configuration Management）與「文件管理」的核心要求，確保所有安全相關的工作產物都能被有效追蹤與版本控制。

**為什麼要這樣算**：

為了實現自動化並減少人為錯誤。當分析報告有多個版本時，手動更新日期很容易出錯或遺漏。透過這個公式，只要版本號被確定（通常是選擇最新的版本），日期就會自動帶入，確保了報告封面資訊的即時性與準確性，這對於後續的審核與追蹤非常重要。

**ASIC 部門注意事項**：

ASIC 設計與驗證團隊需要確保版本歷史紀錄表（`D33:G48`）的維護是準確且即時的。每當完成一次新的 FMEDA 分析並產生新版本時，都必須在此表中新增一筆紀錄，包含正確的版本號與日期。若此表單維護不當，將導致封面頁資訊錯誤，可能在客戶審核或認證過程中引發問題。

### 公式 2：自動選取最新分析版本

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | D3 |
| 公式原文 | `=IF(SUM(D33:D48)=0,"--",MAX(D33:D48))` |
| 應用範圍 | D3 |
| 計算值 | -- |

**參照來源**：

*   `D33:D48`：版本歷史紀錄表中的版本號列表。

**計算邏輯（白話文）**：

這個公式首先加總 `D33:D48` 範圍內所有的版本號。如果總和為 0（代表這個列表是空的），則顯示 `"--"`。否則，它會從 `D33:D48` 這個範圍中找出最大的數值，也就是最新的版本號。

**物理意義**：

此公式的物理意義是「自動化版本選擇」。它確保 FMEDA 報告預設選擇並顯示當前最新的分析版本號，避免使用者需要手動從清單中挑選，從而簡化操作並降低選擇到舊版本的風險。

**標準依據**：

與前一個公式類似，這並非直接來自安全標準的計算要求，而是實現 **ISO 26262-2:2018** 中「組態管理」精神的工程實踐。確保工作產物（FMEDA 報告）總是反映最新的分析狀態，是維持安全生命週期一致性的關鍵。

**為什麼要這樣算**：

為了確保報告的即時性與預設正確性。在一個持續迭代的專案中，分析會不斷更新。將最新版本號自動顯示在封面上，可以讓所有檢視者立即知道這份報告是基於最新的分析結果，避免因參考到過時文件而做出錯誤決策。

**ASIC 部門注意事項**：

ASIC 團隊在維護版本歷史紀錄表時，必須確保版本號是「純數字」且具備可比較性（例如 `1.0`, `1.1`, `2.0`）。如果版本號包含無法被 `MAX` 函數比較的文字（例如 `v1.0-alpha`），此公式將會失效。建議建立明確的版本號命名規則，以確保此自動化功能的正常運作。

### 公式 3：版本歷史紀錄的作者欄位

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | G33 |
| 公式原文 | `=A33` |
| 應用範圍 | G33 到 G48（共 16 個儲存格） |
| 計算值 | 0 |

**參照來源**：

*   `A33`：對應同一橫列的 `A` 欄儲存格，通常是作者或修改者資訊。

**計算邏輯（白話文）**：

這個公式非常直接，就是將儲存格 `G33` 的內容設定為與同一橫列的 `A33` 儲存格完全相同。這個模式被向下複製到 `G34`（等於 `A34`）、`G35`（等於 `A35`），以此類推，直到 `G48`。

**物理意義**：

此公式的物理意義是「資訊的複製與同步」。它將 `A` 欄的作者或修改者資訊，複製到 `G` 欄。在 `VLOOKUP` 的應用中（如公式 1），`G` 欄可能作為回傳值。雖然在此特定範例中，公式 1 回傳的是第 4 欄（F 欄），但這種結構通常是為了讓一個表格同時服務於多種查找需求。

**標準依據**：

這同樣是支援 **ISO 26262-2:2018** 中「文件管理」與「可追溯性」要求的實踐。確保每一次變更都有對應的作者或修改者紀錄，是建立完整變更歷史的基礎。

**為什麼要這樣算**：

為了確保資料的一致性並簡化表格維護。與其在 `A` 欄和 `G` 欄手動輸入兩次相同的作者資訊，不如使用公式進行關聯。這樣一來，當需要更新作者資訊時，只需修改 `A` 欄，`G` 欄就會自動同步，避免了手動修改可能導致的兩邊資訊不一致的問題。

**ASIC 部門注意事項**：

ASIC 團隊成員在更新版本歷史紀錄時，應僅在 `A` 欄填寫或修改作者資訊。應避免直接修改 `G` 欄，因為這樣會破壞公式，導致資料不再同步。團隊應建立共識，僅維護主要的資料來源欄位（`A` 欄），讓公式來處理資料的複製與呈現。
