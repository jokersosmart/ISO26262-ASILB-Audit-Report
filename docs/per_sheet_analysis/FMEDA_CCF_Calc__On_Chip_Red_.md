### 公式 1：額外 β 因子計算

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | E5 |
| 公式原文 | `=C5*F5` |
| 應用範圍 | E5, E6, E7, E8, E14, E15, E16, E17, E19, E20, E21, E22, E23, E24 |
| 計算值 | 5 |

**參照來源**：

*   **C5**: 共同失效群組中的元件數量 (Number of elements in the common cause failure group)。
*   **F5**: 該群組中每個元件的基礎 β 因子 (β-factor for each element)。

**計算邏輯（白話文）**：

此公式將「共同失效群組中的元件數量」(C5) 與「該群組中每個元件的基礎 β 因子」(F5) 相乘，用以計算出由該特定共因事件所貢獻的「額外 β 因子」。這個計算是為了量化一組功能相近或實體位置相近的元件，因為單一根本原因而同時失效的可能性。

**物理意義**：

在功能安全分析中，這個計算結果代表一個特定的「共因失效 (Common Cause Failure, CCF)」事件對系統整體可靠度的影響程度。β 因子是一個機率值，描述了由單一根本原因（例如：電源雜訊、時脈抖動、局部過熱）導致多個元件同時失效的傾向。計算出的「額外 β 因子」越高，表示這個共因群組的失效對整個晶片的安全完整性威脅越大，因為它會同時破壞多個本應相互獨立的安全機制或功能單元。

**標準依據**：

此計算方法基於 **ISO 26262-5:2018, Annex D** 中描述的共因失效分析方法，特別是 β 因子模型 (β-factor model)。該模型是業界廣泛用於評估硬體元件之間相依失效的標準方法之一。β 因子的具體數值通常會參考 **IEC 61508**、**IEC TR 62380** 或 **SN 29500** 等標準中的表格，這些表格根據元件之間的耦合程度（例如，技術、環境、設計上的相似性）提供了經驗值。

**為什麼要這樣算**：

在複雜的 SSD Controller 晶片中，許多邏輯單元並非完全獨立。若只考慮單一元件的隨機硬體失效，而忽略了它們可能因為共享資源（如電源、時脈）或相近的實體佈局而同時失效的風險，將會嚴重低估系統的真實失效率，進而導致對安全風險的誤判。此計算的目的，正是為了將這種「共因失效」的風險予以量化，並將其納入整體的硬體架構度量（如 SPFM、LFM）與 PMHF (Probabilistic Metric for Hardware Failure) 的計算中，以確保對系統安全等級的評估是全面且保守的。

**ASIC 部門注意事項**：

1.  **識別共因群組 (CCG)**：ASIC 設計團隊必須與功能安全工程師緊密合作，在設計早期就系統性地識別出晶片內部所有潛在的共因群組。審查的重點應包含共享電源域 (power domains)、時脈域 (clock domains)、重置信號 (reset signals)、以及在實體佈局上極為接近的關鍵邏輯單元。
2.  **設計多樣性與隔離**：為了降低 β 因子，應在設計中引入多樣性 (Diversity) 與隔離 (Isolation) 的概念。例如，對於冗餘的安全機制，可以考慮使用不同設計的邏輯電路來實現（設計多樣性），或將它們佈置在晶片上不同的區域並由不同的電源供應（實體隔離）。
3.  **提供設計參數依據**：ASIC 團隊需要為 C5（群組中的元件數量）和 F5（基礎 β 因子）的選取提供明確的設計依據。F5 的值取決於耦合因子 (Coupling Factor)，設計團隊應根據 ISO 26262-5 Annex D 的指引，評估並記錄元件之間在設計、環境、溫度等方面的耦合程度，以合理化 F5 的取值。

### 公式 2：額外 β 因子總和

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | E10 |
| 公式原文 | `=SUM(E5:E8)` |
| 應用範圍 | E10, E26 |
| 計算值 | 14 |

**參照來源**：

*   **E5:E8**: 一系列由不同共因群組計算出的「額外 β 因子」。

**計算邏輯（白話文）**：

此公式將 E5 到 E8 儲存格中的數值相加。這些儲存格分別代表了不同共因事件（例如，與看門狗功能相關的元件、與時脈監控相關的元件等）所貢獻的「額外 β 因子」。這個總和代表了所有已識別的、會影響晶片內部冗餘設計的共因失效事件所帶來的累積風險。

**物理意義**：

這個計算結果是「總額外 β 因子」，它量化了晶片上所有考慮的共因失效情境對安全目標的總體威脅。在 FMEDA 分析中，單一的 β 因子描述了特定一組元件的相依性，而這個總和則是將所有這些獨立評估的相依性風險疊加起來，形成一個對整個晶片「共因失效傾向」的宏觀度量。這個數值越高，表示晶片的冗餘設計或安全機制越容易被單一根本原因所集體破壞。

**標準依據**：

此加總計算本身是一個基本的算術運算，但其背後的邏輯遵循 **ISO 26262-5:2018** 的原則，即在進行硬體架構度量（SPFM, LFM）和 PMHF 計算時，必須考慮所有可預見的失效模式，其中就包括了多個共因失效事件的累積效應。雖然標準沒有明確規定一個「總 β 因子」的計算公式，但將各個獨立 CCG 的 β 因子貢獻進行匯總是進行全面性安全評估的必要步驟。

**為什麼要這樣算**：

一個複雜的 SSD Controller 晶片中存在多個功能迥異但又可能相互影響的安全機制。例如，記憶體保護單元 (MPU) 和 ECC 錯誤校正邏輯雖然功能不同，但它們可能共享同一個電源域。為了得到一個能反映整個晶片穩健性的指標，不能只看單一的共因失效風險，而必須將所有已識別的風險源（E5 到 E8）的影響進行累加。這樣才能在後續計算 PMHF 時，有一個全面的、保守的共因失效機率估計值，確保最終的安全評估不會因為忽略了風險的疊加效應而過於樂觀。

**ASIC 部門注意事項**：

1.  **確保 CCG 識別的完整性**：此公式的準確性高度依賴於 E5 到 E8 所代表的共因群組 (CCG) 是否被完整地識別出來。ASIC 設計團隊在進行設計審查 (Design Review) 時，必須與系統工程師和安全分析師合作，確保沒有遺漏任何重要的潛在共因失效源。任何共享的資源，無論是邏輯上還是物理上的，都應該被納入考量。
2.  **避免重複計算**：在定義不同的 CCG 時，需要注意避免將同一個元件或同一個根本原因重複計入不同的群組中，否則會導致 β 因子的總和被高估，從而得出過於悲觀的安全評估結果。每個 CCG 應該基於一個清晰且獨立的共因假設。
3.  **提供設計隔離措施的證據**：如果 ASIC 團隊在設計中採取了特定的隔離措施（例如，為不同的安全功能提供獨立的電源軌或時脈樹），這些措施可以作為降低特定 CCG 之 β 因子的有力證據。設計文檔中應詳細記錄這些隔離設計，並說明其如何降低了元件之間的耦合度。

### 公式 3：最終 β 因子計算

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | E30 |
| 公式原文 | `=E28+E10-E26` |
| 應用範圍 | E30 |
| 計算值 | 47 |

**參照來源**：

*   **E28**: 基礎 β 因子 (Base β-factor)，通常是根據標準或經驗給定的一個初始值。
*   **E10**: 所有已識別的共因群組所貢獻的「額外 β 因子總和」。
*   **E26**: 針對某些特定情況下的「β 因子扣除項」，可能代表某些設計上的改進或冗餘措施所帶來的正面效益。

**計算邏輯（白話文）**：

此公式以一個「基礎 β 因子」(E28) 為起點，加上所有共因事件累積的「額外 β 因子總和」(E10)，最後再減去一個「扣除項」(E26)。這個計算的目的是在一個基準值之上，疊加所有已知的負面影響（共因風險），並扣除已實現的正面影響（風險緩解措施），從而得到一個最終的、能代表整個晶片冗餘設計所面臨的綜合共因失效風險的「結果 β 因子」。

**物理意義**：

這個計算結果，即「結果 β 因子」(Resulting β-factor)，是整個 FMEDA 分析中關於共因失效的核心指標。它代表了在考慮了所有已識別的共因情境以及相關的設計緩解措施之後，一個元件的失效與其冗餘備份元件的失效之間，由共因事件所導致的條件機率。這個數值將直接被用於計算 SPFM (Single-Point Fault Metric) 和 LFM (Latent Fault Metric) 中的多點失效 (Multiple-Point Faults) 部分，並最終影響 PMHF (Probabilistic Metric for Hardware Failure) 的總體計算結果。一個較高的「結果 β 因子」意味著晶片的冗餘設計更容易被單一原因擊潰，安全完整性較低。

**標準依據**：

這個公式的結構是 β 因子模型在實際應用中的一種體現，其計算邏輯符合 **ISO 26262-5:2018, Annex D** 的精神。Annex D 提供了計算 β 因子的框架，允許分析師根據具體的耦合因子（例如，相鄰性、共享資源）來調整 β 值。此公式中的加減法，可以看作是將 Annex D 中定性的耦合因子評估，轉化為一個可量化的計算流程。E28 可視為基準值，E10 是根據分析得出的懲罰項，E26 則是獎勵項。

**為什麼要這樣算**：

為了得到一個既全面又準確的共因失效評估，不能只使用一個固定的、從標準中查表得來的 β 因子。真實世界的晶片設計非常複雜，一個單一的數值無法反映其細微的設計差異。此公式的設計允許進行更精細的風險調整：
1.  從一個公認的**基準值 (E28)** 開始。
2.  系統性地疊加所有透過分析識別出的**負面因素 (E10)**，確保了評估的保守性與全面性。
3.  公平地考慮所有已實施的**正面設計措施 (E26)**，使評估結果能準確反映設計的優點。
這種加減調整的方式，使得最終的 β 因子能夠更真實地反映特定晶片設計的獨特性，而不是一個籠統的行業平均值。

**ASIC 部門注意事項**：

1.  **為 E28 和 E26 的取值提供依據**：ASIC 團隊不僅要分析導致風險增加的因素 (E10)，更要為 E28 (基礎 β 因子) 和 E26 (扣除項) 的取值提供強有力的設計證據。例如，如果宣稱採取了某種高級的隔離技術而獲得了 E26 的扣除，那麼必須有設計文檔、佈局圖或模擬數據來證明該技術的有效性。
2.  **關注 β 因子的敏感度**：ASIC 團隊應了解，最終的「結果 β 因子」對 SPFM 和 LFM 的影響非常顯著。在設計階段，應進行敏感度分析，了解哪些設計決策對 β 因子的影響最大。這有助於在成本和安全之間做出權衡，將資源集中在最能有效降低 β 因子的設計改進上。
3.  **文檔化所有假設**：所有關於 β 因子的計算，特別是 E28 和 E26 這類帶有經驗判斷或設計宣稱的參數，其背後的所有假設、依據和考量都必須被清晰地記錄在案。這在後續的功能安全稽核 (Functional Safety Audit) 中至關重要，稽核員會嚴格審查這些數值的合理性。
