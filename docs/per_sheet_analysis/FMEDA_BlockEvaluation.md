### 公式 1：從 BlockList 複製儲存格 A10

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | A2 |
| 公式原文 | `=BlockList!A10` |
| 應用範圍 | A2 到 A201（共 200 個儲存格） |
| 計算值 | 1 |

**參照來源**：`BlockList` 工作表的 `A10` 儲存格。

**計算邏輯（白話文）**：這個公式直接複製 `BlockList` 工作表 `A10` 儲存格的內容。根據其應用範圍，A3 會複製 B10，A4 會複製 C10，依此類推。這是一個典型的資料對應或資料傳遞操作。

**物理意義**：在功能安全分析中，這代表將 `BlockList` 中定義的組件編號（No.）傳遞到 `BlockEvaluation` 工作表中，用於後續的分析和追蹤。每個數字唯一標識一個硬體區塊或元件。

**標準依據**：這不是一個計算公式，而是一個資料引用操作。其本身沒有直接的標準依據，但它是 FMEDA 分析流程中資料管理和追溯性的一部分，符合 ISO 26262 對於文件化和可追溯性的總體要求。

**為什麼要這樣算**：為了在不同的分析工作表之間保持資料的一致性。透過公式引用，當 `BlockList` 中的資訊更新時，`BlockEvaluation` 工作表中的對應資料會自動更新，避免了手動複製貼上可能導致的錯誤和遺漏。

**ASIC 部門注意事項**：請確保 `BlockList` 中的組件劃分是清晰且有意義的。每個編號對應的硬體區塊應該與設計層次的模組（Module）或子模組（Sub-module）有明確的對應關係，以便將分析結果準確地映射回實際設計中。

### 公式 2：計算總失效率（永久性 + 暫時性）

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | F2 |
| 公式原文 | `=D2+E2` |
| 應用範圍 | F2 到 F201（共 200 個儲存格） |
| 計算值 | 0 |

**參照來源**：本工作表 (`BlockEvaluation`) 的 `D2` 和 `E2` 儲存格。

**計算邏輯（白話文）**：將同一個硬體區塊的「永久性失效率」（Permanent Failure Rate）和「暫時性失效率」（Transient Failure Rate）相加，得到該區塊的總失效率。

**物理意義**：這代表一個硬體區塊在單位時間內可能發生任何類型失效（無論是永久損壞還是暫時性錯誤）的總和機率。這是後續所有安全度量（SPFM, LFM, PMHF）計算的基礎。單位為 FIT (Failure In Time)，即每十億小時發生一次失效。

**標準依據**：ISO 26262-5:2018, Annex D。該附錄描述了如何對硬體元件進行失效率分析，其中總失效率是各個失效模式失效率的總和。

**為什麼要這樣算**：為了全面評估一個組件的可靠性，必須同時考慮永久性失效（例如，電晶體擊穿）和暫時性失效（例如，單粒子翻轉 SEU）。將兩者相加可以得到一個整體的、更保守的失效率估計，用於評估是否滿足安全目標。

**ASIC 部門注意事項**：ASIC 部門提供的 Gate Count 或 Area 資訊是計算 `D2`（永久性失效率）的基礎。同時，對於暫時性失效率 `E2`，需要提供關於晶片製程技術（Technology Node）、使用的 Flip-Flop 類型等資訊，以便根據例如 IEC TR 62380 等標準估算其 SER (Soft Error Rate)。請確保這些數據的準確性。

### 公式 3：計算單點故障度量（SPFM）- 永久性失效

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | BF2 |
| 公式原文 | `=IF(G2=0,"-",1-(AE2/G2))` |
| 應用範圍 | BF2 到 BF201（共 200 個儲存格） |
| 計算值 | #DIV/0! |

**參照來源**：本工作表 (`BlockEvaluation`) 的 `G2` 和 `AE2` 儲存格。

**計算邏輯（白話文）**：這個公式計算單點故障度量 (Single-Point Fault Metric, SPFM)。計算方法是 1 減去「未被安全機制覆蓋的單點/殘餘故障的失效率 (`AE2`)」除以「所有與安全相關的總失效率 (`G2`)」。如果總失效率為 0，則顯示 ` - `。

**物理意義**：SPFM 代表了整個系統對於「單點故障」的穩健性（robustness）。一個高的 SPFM 值意味著大部分可能導致直接違反安全目標的單點故障，都已經被安全機制所覆蓋。這個度量僅針對單點故障（Single-Point Faults）和殘餘故障（Residual Faults）。

**標準依據**：ISO 26262-5:2018, 8.4.4。公式定義為 SPFM = 1 - (Σλ_SPF + Σλ_RF) / Σλ_total。在此 Excel 中，`AE2` 代表了 Σ(λ_SPF + λ_RF)，而 `G2` 代表了 Σλ_total（僅考慮永久性失效）。

**為什麼要這樣算**：這是 ISO 26262 標準規定的硬體架構度量之一。汽車安全完整性等級 (ASIL) 對 SPFM 有明確的要求（例如，ASIL B 要求 SPFM ≥ 90%）。計算此度量是為了證明設計的硬體架構滿足目標 ASIL 等級的要求。

**ASIC 部門注意事項**：為了提高 SPFM，ASIC 部門需要實作有效的安全機制（Safety Mechanisms）來覆蓋潛在的單點故障。例如，為重要的暫存器或記憶體增加 ECC 或 Parity 檢查。安全機制的診斷覆蓋率 (Diagnostic Coverage, DC) 會直接影響 `AE2` 的大小，進而決定 SPFM 的值。設計團隊必須與安全分析團隊密切合作，以確保所設計的安全機制能有效降低殘餘故障率。

### 公式 4：計算安全失效率（永久性）

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | P2 |
| 公式原文 | `=SUMIFS(FMEDA!$HB$4:$HB$1653,FMEDA!$B$4:$B$1653,B2,FMEDA!$HJ$4:$HJ$1653,"N")` |
| 應用範圍 | P2 到 P201（共 200 個儲存格） |
| 計算值 | 0 |

**參照來源**：`FMEDA` 工作表的 `$HB$4:$HB$1653`、`$B$4:$B$1653` 和 `$HJ$4:$HJ$1653` 範圍，以及本工作表的 `B2` 儲存格。

**計算邏輯（白話文）**：此公式在 `FMEDA` 工作表中查找與當前硬體區塊（由 `B2` 中的名稱標識）相關的所有失效模式。它會加總所有被歸類為「安全」（Safe, 'S'）且非「暫時性」（'N' for No, is not transient）的永久性失效模式的失效率（位於 `HB` 欄）。

**物理意義**：這個值代表了特定硬體區塊中，所有那些雖然會發生，但不會導致安全目標被違反的「永久性」故障的總失效率。這些被認為是「安全故障」（Safe Failures）。例如，一個像素點永久失效，在大多數顯示應用中可能被視為安全故障。

**標準依據**：ISO 26262-5:2018, C.2.3。標準定義了不同的故障類別，包括安全故障（Safe Fault）。FMEDA 的目的之一就是將總失效率分解到各個類別中。`SUMIFS` 是一種實現這種分類加總的實用方法。

**為什麼要這樣算**：為了從總失效率中分離出「安全」的部分。區分安全與不安全的故障是功能安全分析的核心。只有「不安全」的故障才需要被安全機制所覆蓋或控制。這個計算是量化硬體設計固有安全性的第一步。

**ASIC 部門注意事項**：ASIC 設計團隊在進行功能設計時，需要與安全團隊合作，識別哪些潛在的故障模式可以被歸類為「安全」。這通常涉及到對晶片在整車系統中具體應用的理解。例如，某個內部狀態機的單一位元錯誤如果不會對輸出功能產生任何影響，則可能被歸類為安全故障。

### 公式 5：計算潛在故障度量（LFM）- 永久性失效

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | BI2 |
| 公式原文 | `=IF(OR(G2=0,G2-AE2=0),"-",1-(AZ2/(G2-AE2)))` |
| 應用範圍 | BI2 到 BI201（共 200 個儲存格） |
| 計算值 | #DIV/0! |

**參照來源**：本工作表 (`BlockEvaluation`) 的 `G2`、`AE2` 和 `AZ2` 儲存格。

**計算邏輯（白話文）**：這個公式計算潛在故障度量 (Latent Fault Metric, LFM)。計算方法是 1 減去「未被安全機制覆蓋的潛在故障的失效率 (`AZ2`)」除以「所有與安全相關的總失效率 (`G2`) 減去單點和殘餘故障率 (`AE2`)」。如果分母為 0，則顯示 ` - `。

**物理意義**：LFM 代表了整個系統對於「潛在故障」的穩健性。潛在故障是指一個多點故障（Multiple-Point Fault）中的第一個故障，它本身不會違反安全目標，但會使系統處於一種更容易因第二個故障而失效的狀態。一個高的 LFM 值意味著大部分的潛在故障都能被安全機制所偵測到。

**標準依據**：ISO 26262-5:2018, 8.4.5。公式定義為 LFM = 1 - (Σλ_MPF,latent) / (Σλ_total - Σλ_SPF - Σλ_RF)。在此 Excel 中，`AZ2` 代表了 Σλ_MPF,latent，而 `G2-AE2` 代表了分母。

**為什麼要這樣算**：這是 ISO 26262 標準規定的另一個硬體架構度量。ASIL 對 LFM 也有明確的要求（例如，ASIL B 要求 LFM ≥ 60%）。計算此度量是為了證明設計的硬體架構能夠有效地防止由多個獨立故障累積導致的風險。

**ASIC 部門注意事項**：為了提高 LFM，ASIC 部門需要實作能夠檢測潛在故障的安全機制。這通常比檢測單點故障更具挑戰性。例如，可以使用線上自測試（Online Self-Test）機制，定期檢查安全機制本身是否正常工作，或者檢查是否存在一些隱蔽的硬體故障。這些測試的診斷覆蓋率 (DC) 直接影響 `AZ2` 的大小，從而決定 LFM 的值。
