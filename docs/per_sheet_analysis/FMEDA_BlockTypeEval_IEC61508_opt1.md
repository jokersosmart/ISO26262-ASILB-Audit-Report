### 公式 29：條件判斷

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | `AC2` |
| 公式原文 | `=IF(SUM(N2,Q2)=0,"Not calculated",(N2+T2)/(N2+Q2))` |
| 應用範圍 | AC2 到 AC9（共 7 個儲存格） |
| 計算值 | Not calculated |

**參照來源**：`N2`、`Q2`、`N2`、`T2`、`N2`、`Q2`

**計算邏輯（白話文）**：首先檢查『安全失效』（N2）和『危險失效』（Q2）的永久性失效率總和是否為零。如果是零，則顯示『Not calculated』；如果不為零，則計算（『安全失效』+『危險已偵測失效』）除以（『安全失效』+『危險失效』）的比率。

**物理意義**：這個公式在計算單點故障度量（Single-Point Fault Metric, SPFM）的粗估值。SPFM衡量的是安全機制能夠覆蓋單點故障的有效程度。分母是所有可能導致單點故障的總失效率（安全失效+危險失效），分子是其中不會導致違反安全目標的失效率（安全失效+危險已偵測失效）。

**標準依據**：公式直接源自 ISO 26262-5 章節 8.4.4 中對 SPFM 的定義：SPFM = (λ_S + λ_RF + λ_MPF_D) / (λ_S + λ_RF + λ_MPF)。在此簡化模型中，λ_S 對應 N2，λ_MPF_D 對應 T2，λ_MPF 對應 Q2。

**為什麼要這樣算**：SPFM 是評估硬體架構是否滿足特定 ASIL 等級的關鍵指標之一。例如，ASIL B 要求 SPFM ≥ 90%。透過這個計算，可以快速評估當前設計的安全機制覆蓋率是否達標。

**ASIC 部門注意事項**：ASIC 部門需要重點關注如何提升『危險已偵測失效』（T2）的比例，這通常是透過增加安全機制（如 ECC、Parity Check、冗餘等）來實現的。此公式的結果直接反映了所設計的安全機制的有效性。如果 SPFM 不達標，則需要重新審視或增加安全設計。
