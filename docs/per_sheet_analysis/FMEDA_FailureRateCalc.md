# FMEDA Analysis Report: FailureRateCalc

This report provides a detailed analysis of the Excel formulas found in the 'FailureRateCalc' sheet. Each formula is broken down to explain its calculation logic, physical meaning in the context of functional safety, the relevant standards it's based on, and important considerations for the ASIC design team.
'''### Formula 1: Average Calculation

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | AO3 |
| 公式原文 | `=AVERAGE(50,60)` |
| 應用範圍 | AO3 |
| 計算值 | 55 |

**參照來源**：None

**計算邏輯（白話文）**：此公式計算 50 和 60 的平均值。

**物理意義**：此計算可能代表初步的、或是簡化的失效率估算，可能用於比較不同設計方案的初始評估。在沒有詳細資料時，使用平均值是一種常見的作法。

**標準依據**：直接的平均值計算並無特定標準，但失效率的估算方法可參考 ISO 26262-5, Annex D。

**為什麼要這樣算**：在設計初期，可能缺乏詳細的元件資料，因此使用一個估計的平均值來進行初步的分析，有助於快速評估設計方向。

**ASIC 部門注意事項**：ASIC 部門應了解這個平均值的來源，並在後續設計階段，用更精確的數據（例如來自 IP 供應商的數據或內部歷史數據）來取代這個初步估算。

### Formula 2: Simple Average Calculation

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | AQ3 |
| 公式原文 | `=(0.021+0.026+0.032)/3` |
| 應用範圍 | AQ3 |
| 計算值 | 0.026333333333333334 |

**參照來源**：無，為寫死 (Hardcoded) 的數值。

**計算邏輯（白話文）**：此公式計算三個給定數值（0.021、0.026、0.032）的算術平均值。

**物理意義**：這三個數值很可能是某個電子元件或 IP 在不同參考資料、不同供應商或不同操作條件下的基本失效率（Base Failure Rate），單位通常是 FIT（Failures In Time，每十億小時的失效次數）。計算平均值是為了在缺乏單一權威數據時，得到一個更具代表性的綜合失效率估計值。

**標準依據**：這種取平均值的作法本身並非標準條文，而是一種工程實踐。然而，其背後的失效率估算概念依據 **IEC TR 62380** 或 **SN 29500**，這些標準提供了電子元件失效率的預測模型與數據。

**為什麼要這樣算**：在功能安全分析初期，可能會從多個數據手冊或供應商那裡獲得不同的失效率數據。為了不偏向任何單一數據源，採用平均值可以平滑掉極端值，得到一個相對中立且合理的初步估算，用於後續的 FMEDA 分析。

**ASIC 部門注意事項**：ASIC 部門需要追溯這三個數值的來源。它們是來自公開的數據手冊（如 IEC TR 62380）、特定供應商的數據，還是公司內部的歷史數據？理解數據來源的可靠性至關重要。如果這是某個關鍵 IP 的失效率，應優先採用 IP 供應商提供的官方數據，並確認其估算時所採用的製程、電壓和溫度等條件是否與本專案相符。

### Formula 3: Failure Rate Scaling Calculation

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | AR3 |
| 公式原文 | `=AO3/AO4*AP3/AP4*AR4` |
| 應用範圍 | AR3 |
| 計算值 | 0.007711452685324477 |

**參照來源**：`AO3`, `AO4`, `AP3`, `AP4`, `AR4`

**計算邏輯（白話文）**：此公式將一個基礎值 `AR4` 乘以兩個比例因子 (`AO3/AO4` 和 `AP3/AP4`) 來得到一個調整後的值。

**物理意義**：此公式極有可能是在對一個已知的參考失效率 (`AR4`) 進行調整，以使其適用於當前的特定情況。  
- `AR4` 可能是一個從數據手冊（如 IEC TR 62380）中查到的、針對某個參考晶片或IP的基礎失效率（Base FIT Rate）。  
- `AO3/AO4` 很可能是一個**邏輯閘數 (Gate Count) 的比例因子**。例如，`AO4` 是參考晶片的總閘數，而 `AO3` 是當前設計中所使用部分的閘數。此比例用於將參考失效率縮放到符合當前設計的規模。  
- `AP3/AP4` 可能是一個**壓力因子 (Stress Factor) 的比例**，例如溫度或電壓。`AP4` 是參考條件下的壓力值，`AP3` 是當前設計的實際工作壓力值。此比例用於根據實際工作環境來修正失效率。

**標準依據**：這種按比例縮放 (Scaling) 的方法是失效率預測中的常用技術，其概念根植於 **IEC TR 62380** 和 **SN 29500**。這些標準的失效率模型通常包含基礎失效率，並透過多個因子（如溫度、電壓、製程複雜度）進行調整。

**為什麼要這樣算**：很難為每一個設計都找到完全匹配的失效率數據。因此，從一個相似的、有數據的參考設計出發，並根據自己設計的具體參數（如規模、工作環境）進行按比例調整，是一種兼顧準確性和實用性的工程方法。

**ASIC 部門注意事項**：ASIC 部門必須仔細核對此公式中的每一個參數。  
1.  **確認 `AR4` 的來源**：這個參考失效率是來自哪個晶片或哪個數據手冊？其參考的製程、電壓、溫度條件是什麼？  
2.  **驗證 Gate Count `AO3` 和 `AO4`**：確保 `AO3` 準確反映了當前設計的邏輯規模。`AO4` 作為參考規模，也需要確認其準確性。  
3.  **審查壓力因子 `AP3` 和 `AP4`**：確認 `AP3` 是否準確代表了晶片在目標應用中的最壞工作條件（Worst-case operating condition）。如果這是溫度因子，它應該是 Tj (Junction Temperature) 而非環境溫度。

### Formula 7: Average Failure Rate Calculation

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | AR4 |
| 公式原文 | `=AVERAGE(98.48/8500000000,122/10000000000,90/8000000000,113.31/10300000000,74.13/6900000000,83.27/6900000000,112/8500000000,112/8500000000)*1000000` |
| 應用範圍 | AR4 |
| 計算值 | 0.011900173575762089 |

**參照來源**：無，為寫死 (Hardcoded) 的數值。

**計算邏輯（白話文）**：此公式計算八個分數的平均值，然後將結果乘以一百萬。每個分數都是一個小數除以一個非常大的數。

**物理意義**：此公式正在計算來自八個不同數據源的平均失效率。每個分數（例如 `98.48/8500000000`）很可能代表 `觀察到的失效數 / 總運行時間`。將這些比率平均起來，可以得到一個更穩健的失效率估計。最後乘以 1,000,000 可能是為了將單位從「每小時失效次數」轉換為「每百萬小時失效次數」。

**標準依據**：從原始數據計算失效率的方法是可靠性工程的基本原則，並被 **IEC TR 62380** 和 **SN 29500** 等標準所採納。這些標準提供了模型和參考數據，但計算率的基本方法是首要原則。對多個來源進行平均是一種常見的工程實踐，以獲得更可靠的估計。

**為什麼要這樣算**：當可靠性數據可從多個現場研究、可靠性測試或不同的產品變體中獲得時，使用此方法。對它們進行平均可提供一個綜合的失效率，該失效率對任何單個數據集的特殊性都不那麼敏感。縮放因子使最終數字更易於管理和比較。

**ASIC 部門注意事項**：
1.  **數據源調查**：最關鍵的任務是了解這八對數字的來源。它們是來自現場反饋、加速壽命測試還是其他來源？它們代表哪些特定的產品或組件？
2.  **單位驗證**：必須確認分子和分母的物理意義。分母真的是小時，還是像上電週期或寫入數據之類的其他東西？這對於正確解釋最終結果至關重要。
3.  **縮放因子基本原理**：為什麼結果要乘以 1,000,000？如果目標單位是 FIT（每 10^9 小時的失效次數），則乘數應為 1,000,000,000。ASIC 團隊需要澄清此計算的預期輸出單位。

**標準依據**：這種取平均值的作法本身並非標準條文，而是一種工程實踐。然而，其背後的失效率估算概念依據 **IEC TR 62380** 或 **SN 29500**，這些標準提供了電子元件失效率的預測模型與數據。

**為什麼要這樣算**：在功能安全分析初期，可能會從多個數據手冊或供應商那裡獲得不同的失效率數據。為了不偏向任何單一數據源，採用平均值可以平滑掉極端值，得到一個相對中立且合理的初步估算，用於後續的 FMEDA 分析。

**ASIC 部門注意事項**：ASIC 部門需要追溯這七個數值的來源。它們是來自公開的數據手冊（如 IEC TR 62380）、特定供應商的數據，還是公司內部的歷史數據？理解數據來源的可靠性至關重要。如果這是某個關鍵 IP 的失效率，應優先採用 IP 供應商提供的官方數據，並確認其估算時所採用的製程、電壓和溫度等條件是否與本專案相符。

### Formula 11: Average Failure Rate Calculation

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | AR5 |
| 公式原文 | `=AVERAGE(398/18000000000,96.4/4300000000,96.72/5500000000,89.23/4300000000,94/5300000000)*1000000` |
| 應用範圍 | AR5 |
| 計算值 | 0.02012043643100598 |

**參照來源**：無，為寫死 (Hardcoded) 的數值。

**計算邏輯（白話文）**：此公式計算五個分數的平均值，然後將結果乘以一百萬。每個分數都是一個小數除以一個非常大的數。

**物理意義**：此公式正在計算來自五個不同數據源的平均失效率。每個分數（例如 `398/18000000000`）很可能代表 `觀察到的失效數 / 總運行時間`。將這些比率平均起來，可以得到一個更穩健的失效率估計。最後乘以 1,000,000 可能是為了將單位從「每小時失效次數」轉換為「每百萬小時失效次數」。

**標準依據**：從原始數據計算失效率的方法是可靠性工程的基本原則，並被 **IEC TR 62380** 和 **SN 29500** 等標準所採納。這些標準提供了模型和參考數據，但計算率的基本方法是首要原則。對多個來源進行平均是一種常見的工程實踐，以獲得更可靠的估計。

**為什麼要這樣算**：當可靠性數據可從多個現場研究、可靠性測試或不同的產品變體中獲得時，使用此方法。對它們進行平均可提供一個綜合的失效率，該失效率對任何單個數據集的特殊性都不那麼敏感。縮放因子使最終數字更易於管理和比較。

**ASIC 部門注意事項**：
1.  **數據源調查**：最關鍵的任務是了解這五對數字的來源。它們是來自現場反饋、加速壽命測試還是其他來源？它們代表哪些特定的產品或組件？
2.  **單位驗證**：必須確認分子和分母的物理意義。分母真的是小時，還是像上電週期或寫入數據之類的其他東西？這對於正確解釋最終結果至關重要。
3.  **縮放因子基本原理**：為什麼結果要乘以 1,000,000？如果目標單位是 FIT（每 10^9 小時的失效次數），則乘數應為 1,000,000,000。ASIC 團隊需要澄清此計算的預期輸出單位。
