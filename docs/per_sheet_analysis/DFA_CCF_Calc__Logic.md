
### 公式 1：CCF 分數 X 計算

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | F5 |
| 公式原文 | `=B5*H5` |
| 應用範圍 | F5 到 F49（共 29 個儲存格） |
| 計算值 | 0 |

**參照來源**：儲存格 B5、H5

**計算邏輯（白話文）**：將儲存格 B5 的數值與儲存格 H5 的數值相乘，得到一個結果 X。

**物理意義**：此公式用於計算「共因失效」（Common Cause Failure, CCF）分析中的一個權重分數 X。在相依失效分析（Dependent Failure Analysis, DFA）的框架下，這個計算代表了某個特定的「耦合因子」（Coupling Factor）的量化貢獻。儲存格 B5 通常代表一個定性的評分（例如，組件之間的物理隔離程度），而 H5 則可能是一個相關的定量參數或另一個權重。X 的計算結果是整體 CCF 評分的一部分。

**標準依據**：CCF 的分析方法論主要基於 **ISO 26262-9:2018, Annex E**，該附錄提供了相依失效分析的詳細指引。此處的具體評分系統（例如 X 和 Y 的計算）可能是公司根據標準原則自定義的實作方法，用以量化評估 CCF 的風險。

**為什麼要這樣算**：為了量化單一失效原因導致多個組件同時失效的風險。透過將代表不同耦合機制（例如，物理鄰近性、共享訊號、共享電源）的因子相乘，可以得到一個能夠代表 CCF 事件發生可能性的分數。這個分數有助於識別設計中的薄弱環節，並採取對應的緩解措施。

**ASIC 部門注意事項**：B5 和 H5 的數值是此計算的關鍵輸入，ASIC 設計團隊必須理解是哪些具體的設計特徵影響了這些數值。例如，如果 B5 代表實體間距，佈局（Layout）工程師必須確保冗餘區塊之間滿足所要求的物理隔離距離。如果 H5 與共享資源（如時脈樹、重置訊號）有關，則必須審視設計，以最小化這種相依性，或確保該共享資源本身具備更高的可靠性，例如透過三重冗餘（TMR）設計來加固。

### 公式 2：CCF 分數 Y 計算

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | G5 |
| 公式原文 | `=C5*H5` |
| 應用範圍 | G5 到 G49（共 27 個儲存格） |
| 計算值 | 0 |

**參照來源**：儲存格 C5、H5

**計算邏輯（白話文）**：將儲存格 C5 的數值與儲存格 H5 的數值相乘，得到一個結果 Y。

**物理意義**：此公式與前一個公式類似，用於計算 CCF 分析中的另一個權重分數 Y。在 DFA 的脈絡下，X 和 Y 通常代表不同性質的耦合因子。例如，X 可能代表「物理鄰近性」的影響，而 Y 可能代表「設計相似性」的影響。將這些不同的維度分開計算，可以更細緻地評估不同耦合來源的風險貢獻。

**標準依據**：同樣基於 **ISO 26262-9:2018, Annex E** 的相依失效分析原則。具體的 X、Y 評分系統是公司內部為了實踐標準要求而發展出的量化模型。標準本身提供框架和指導原則，但未規定具體的計算公式。

**為什麼要這樣算**：為了從不同維度評估共因失效的風險。將不同的耦合因子（如物理、設計、電氣）分開評分再加總，可以讓分析更全面。如果只用單一指標，可能會忽略某些特定但重要的相依性。例如，兩個功能區塊在晶片上物理位置很遠（低 X 分數），但它們共享了相同的設計邏輯（高 Y 分數），這種情況下，它們仍然有很高的 CCF 風險。

**ASIC 部門注意事項**：ASIC 團隊需要清楚 C5 和 H5 所代表的設計屬性。如果 C5 代表「設計相似度」，那麼在設計冗餘安全機制時，應盡可能採用相異性設計（Design Diversity）。例如，一個安全監控電路和其監控的主要功能電路，應由不同的工程師、使用不同的設計方法或甚至是不同的邏輯庫來實現，以降低兩者因共同的設計缺陷而同時失效的風險。

### 公式 3：耦合因子評分總和 (維度1)

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | B51 |
| 公式原文 | `=SUM(B5:B49)` |
| 應用範圍 | B51 |
| 計算值 | 62.5 |

**參照來源**：儲存格範圍 B5:B49

**計算邏輯（白話文）**：將 B5 到 B49 這一整列的數值全部加總起來。

**物理意義**：此公式計算的是在相依失效分析中，某一個評分維度（例如，與「物理隔離」相關的所有子項目得分）的總和。這個總分本身不是最終的風險指標，而是一個中間計算值，它匯總了所有被評估項目在該維度下的原始分數，用於後續更複雜的 CCF 總分計算。

**標準依據**：這是一個基本的加總運算，用於數據匯總。其計算目的服務於 **ISO 26262-9:2018** 中對相依失效的系統性分析要求，但公式本身並非標準直接定義。

**為什麼要這樣算**：為了將分散在多個項目中的同類評分進行整合，得到一個該維度的總體評分基礎。在進行複雜的風險評估時，通常需要先將各個子項目的貢獻累加，才能進行下一步的加權或組合運算。這一步是數據標準化和簡化的過程。

**ASIC 部門注意事項**：ASIC 團隊需要確保 B5 到 B49 中的每一個評分都經過仔細審核且有據可依。因為這個總和值會直接影響最終的 CCF 評分，任何一個子項目的評分錯誤都將導致最終結果的偏差。設計、佈局和驗證團隊應共同確認這些評分的準確性。

### 公式 7：加權 CCF 分數 X 總和

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | F51 |
| 公式原文 | `=SUM(F5:F49)` |
| 應用範圍 | F51 |
| 計算值 | 0 |

**參照來源**：儲存格範圍 F5:F49

**計算邏輯（白話文）**：將 F5 到 F49 這一整列的數值（即所有子項目的加權分數 X）全部加總起來。

**物理意義**：此公式計算的是所有被評估項目在第一個維度（由 X 代表）上的加權總分。前面的公式 `=B5*H5` 計算了單個項目的 X 分數，而這個 SUM 公式則是將所有這些單項分數匯總，得到一個代表該維度總體 CCF 貢獻的數值。這個總分是計算最終 CCF 風險評分的核心輸入之一。

**標準依據**：這是一個數據匯總步驟，服務於 **ISO 26262-9:2018, Annex E** 的相依失效分析框架。公式本身是基本的數學運算，但其目的是為了實現標準所要求的系統性評估。

**為什麼要這樣算**：為了得到一個能夠代表整個系統在某一個特定耦合維度（例如「物理鄰近性」）下的總體風險分數。單個項目的 X 分數只反映局部情況，將它們全部加總才能從宏觀層面評估該耦合維度的整體影響，並將其與其他維度（如 Y 分數總和）進行比較或組合。

**ASIC 部門注意事項**：由於這個總和值直接來自 F5:F49 的計算結果，ASIC 團隊需要關注的是導致 X 分數產生的根本原因，即 B 列和 H 列的評分。如果 F51 的總分過高，團隊需要回溯到 F 列中數值最高的那些子項，分析其對應的 B 列和 H 列評分來源，並從設計或佈局上採取措施降低這些評分，從而降低 F51 的總分。

### 公式 10：CCF 基礎總分 S 計算

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | B53 |
| 公式原文 | `=(F51+G51)` |
| 應用範圍 | B53 |
| 計算值 | 0 |

**參照來源**：儲存格 F51、G51

**計算邏輯（白話文）**：將兩個加權 CCF 分數的總和（F51 和 G51）相加，得到一個基礎總分 S。

**物理意義**：此公式計算的是一個未經調整的、基礎的 CCF 總分 S。它簡單地將代表兩個不同耦合維度（例如「物理鄰近性」和「設計相似性」）的加權總分相加，得到一個初步的風險評估值。這個分數 S 綜合了兩個主要維度的影響，但可能尚未考慮其他更複雜的交互作用或調整因子。

**標準依據**：這個加總步驟是公司自定義 CCF 量化模型的一部分，旨在整合來自不同耦合維度的風險貢獻。其背後的原則符合 **ISO 26262-9:2018, Annex E** 對於系統性分析的要求，即需要全面考慮所有潛在的相依性來源。

**為什麼要這樣算**：為了將來自不同耦合維度的量化風險指標合併成一個單一的、易於比較的基礎分數。透過將 F51（維度 X 的總貢獻）和 G51（維度 Y 的總貢獻）相加，可以得到一個初步的、全面的 CCF 風險快照。這個基礎分數 S 可以作為後續更詳細分析的起點，或者在簡化的模型中直接作為最終評估的依據。

**ASIC 部門注意事項**：ASIC 團隊應該將 S 視為一個關鍵的設計健康指標。如果 S 的值超過了預設的閾值，這意味著晶片在物理設計或邏輯設計層面存在較高的共因失效風險。團隊需要回溯分析是 F51 還是 G51（或兩者）對 S 的貢獻更大，並針對性地採取改進措施。例如，如果 G51（設計相似性）是主要貢獻者，則應優先考慮引入設計相異性措施。

### 公式 11：CCF 調整後總分 SD 計算

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | B57 |
| 公式原文 | `=F51*(B55+1)+G51` |
| 應用範圍 | B57 |
| 計算值 | 0 |

**參照來源**：儲存格 F51、B55、G51

**計算邏輯（白話文）**：將第一個加權 CCF 分數總和（F51）乘以一個調整因子（B55 的值加 1），然後再加上第二個加權 CCF 分數總和（G51），得到一個調整後的總分 SD。

**物理意義**：此公式計算的是一個經過調整的 CCF 總分 SD。與基礎總分 S 不同，SD 引入了一個調整因子 Z（儲存在 B55），用來對第一個維度（X 維度，由 F51 代表）的影響進行非線性放大。這個調整因子 Z 可能代表某個特殊的、影響力巨大的耦合機制（例如，共享的關鍵電源軌）。當 Z > 0 時，X 維度的風險貢獻會被不成比例地放大，反映了其在特定情況下的更高危險性。

**標準依據**：這是一個公司自定義的高級 CCF 量化模型。這種引入調整因子的做法，體現了 **ISO 26262** 功能安全分析中「專家判斷」（Expert Judgment）和「Tailoring」（剪裁）的精神。標準提供了框架，而公司根據其產品的具體特性和歷史經驗，對評估模型進行微調，使其更貼近實際情況。

**為什麼要這樣算**：為了更精確地反映不同耦合維度之間可能存在的非對稱影響。在某些情況下，某個維度的風險（如 F51）可能會因為另一個因素（如 B55）的存在而被急遽放大。簡單的線性相加（如計算 S）無法捕捉這種交互作用。例如，如果 F51 代表物理鄰近性，而 B55 代表「共享同一個散熱不良的封裝」，那麼物理上靠近的模組會因為過熱問題而具有不成比例的更高同時失效風險。這個公式試圖量化這種放大效應。

**ASIC 部門注意事項**：ASIC 團隊必須極度關注調整因子 Z（B55）的物理意義和設定依據。這個因子是整個 CCF 評估中的「槓桿點」，其數值的微小變化可能導致最終評分 SD 的巨大變動。團隊需要提供強有力的證據來支持 B55 的取值，例如熱模擬分析、電源完整性分析或壓力測試數據。如果 SD 分數過高，除了檢討 F51 和 G51，也必須審視是否存在可以降低 Z 值的設計方案。

### 公式 12：β-因子初估

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | B59 |
| 公式原文 | `=IF(B53<45, 5, IF(B53<70, 2, IF(B53<120, 1, 0.5)))` |
| 應用範圍 | B59, B61 |
| 計算值 | 5 |

**參照來源**：儲存格 B53

**計算邏輯（白話文）**：這是一個多層條件判斷。它根據基礎總分 S（儲存在 B53）的數值，返回一個對應的 β-因子（beta factor）的初步估計值 b。具體邏輯是：
- 如果 S 小於 45，則 b 為 5。
- 否則，如果 S 小於 70，則 b 為 2。
- 否則，如果 S 小於 120，則 b 為 1。
- 如果以上條件都不滿足（即 S 大於等於 120），則 b 為 0.5。

**物理意義**：此公式將連續的 CCF 基礎總分 S 映射到一個離散的、代表 β-因子百分比的數值 b。β-因子是功能安全中一個極其重要的參數，它量化了在一個冗餘系統中，由單一原因導致多個通道同時失效的概率佔總失效率的比例。例如，β = 2% 意味著在所有單通道失效事件中，有 2% 的事件是由於一個共因導致的，並且這個原因同時也讓其他通道失效了。這裡的 b 值（5, 2, 1, 0.5）很可能代表 β-因子的百分比，即 5%、2%、1% 和 0.5%。這個公式的物理意義是：CCF 總分 S 越高，代表設計中的耦合性越強，因此 β-因子也越高，系統的冗餘效益就越差。

**標準依據**：β-因子的概念和應用在 **ISO 26262-5:2018, Annex D** 和 **IEC 61508** 中有詳細描述。然而，將一個公司自定義的 CCF 分數 S 直接映射到 β-因子的具體查閱表（Look-up Table）是公司內部的實踐。標準提供了 β-因子的典型值範圍（例如，對於 ASIL D，β 通常要求小於 1% 或 2%），但沒有規定必須如何從設計參數中推導出它。這個 IF 公式是公司根據其經驗和對標準的理解，建立的一個從定性/半定量分析到定量度量（β-因子）的橋樑。

**為什麼要這樣算**：為了將複雜、多維度的 CCF 分析結果（分數 S）轉化為一個可以直接用於硬體架構度量（Hardware Architectural Metrics）計算的標準參數（β-因子）。SPFM（Single-Point Fault Metric）和 LFM（Latent Fault Metric）的計算都需要 β-因子作為輸入。透過這個公式，設計團隊可以快速地從他們的 DFA 分析結果中，得到一個對 β-因子的初步估計，從而評估其設計是否可能滿足 ASIL 等級的要求。

**ASIC 部門注意事項**：ASIC 團隊需要將這個公式視為設計目標的直接轉換器。例如，如果目標 ASIL 等級要求 β-因子不大於 1%，那麼根據這個公式，團隊必須確保他們設計的 CCF 基礎總分 S（B53）必須小於 70。這為設計和佈局團隊提供了一個非常具體的、可操作的目標。他們在進行物理隔離、增加設計相異性、減少共享資源等所有 CCF 緩解措施時，其最終目的就是要將分數 S 壓低到目標範圍內。這個公式將抽象的安全目標轉化為了具體的設計約束。
