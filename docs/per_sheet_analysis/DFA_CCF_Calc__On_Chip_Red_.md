### 公式 1：計算額外 β 因子

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | E5 |
| 公式原文 | `=C5*F5` |
| 應用範圍 | E5, E6, E7, E8, E14, E15, E16, E17, E19, E20, E21, E22, E23, E24 (共 14 個儲存格) |
| 計算值 | 0 |

**參照來源**：

此公式參照同一工作表中的 `C5` 和 `F5` 儲存格。根據上下文（E欄標題為「additional β-factor [%]」），`C5` 很可能是一個權重因子（Weighting Factor），而 `F5` 則是一個與特定共因失效（Common Cause Failure, CCF）相關的基礎分數或參數。

**計算邏輯（白話文）**：

將儲存格 `C5` 的數值與儲存格 `F5` 的數值相乘，得到一個額外的 β 因子貢獻值。

**物理意義**：

此計算結果代表由特定設計選擇或環境因素所引發的、額外的共因失效（CCF）耦合因子（β-factor）的量化數值。β 因子是用於衡量安全機制對於控制 CCF 有效性的指標，較高的 β 因子意味著系統對共因失效的易感性更高。此處的「額外」β 因子，根據其上下文描述，特別指向因採用晶片內建看門狗（On-Chip Watchdog）等設計決策，而非外部獨立監控單元，所帶來的額外風險貢獻。

**標準依據**：

此計算方法基於 **ISO 26262-5:2018, Annex D** 中關於相依性失效分析（Dependent Failure Analysis, DFA）的指導原則。該附錄詳細說明了如何識別潛在的共因失效，並透過 β 因子模型進行量化分析。具體的權重因子（如 C5）和基礎分數（如 F5）的選取，通常參考該附錄中的表格 D.2 至 D.7，這些表格針對不同的耦合因素（如物理鄰近性、共享資源等）提供了建議值。

**為什麼要這樣算**：

為了系統性地量化特定設計決策對整體功能安全的影響。在功能安全分析中，必須識別並評估所有可能導致多個元件同時失效的根本原因。將監控電路（如看門狗）整合在同一晶片上，雖然可以節省成本和面積，但可能會引入新的共因失效風險（例如，整個晶片因共同的電源或時脈問題而失效）。此公式透過將一個抽象的設計決策（內建看門狗）轉換為一個具體的數學值（額外的 β 因子），使得工程師能夠在後續的 PMHF（Probabilistic Metric for Hardware Failure）計算中，精確評估其對安全目標的影響。

**ASIC 部門注意事項**：

ASIC 設計團隊必須充分理解此計算背後的安全假設。`C5` 和 `F5` 的數值並非隨意設定，而是基於具體的設計架構。團隊需要向功能安全工程師提供詳細的設計資訊，包括：

1.  **資源共享**：看門狗電路與被監控的功能單元是否共享時脈樹、電源軌、重置訊號或基板（Substrate）？
2.  **物理佈局**：這些單元在晶片上的物理距離有多近？是否有隔離結構（如 Guard Rings）？
3.  **設計多樣性**：看門狗的設計與核心功能單元的設計是否存在差異（例如，使用不同的邏輯單元庫或合成策略）？

任何可能導致共因失效的設計細節，都會影響 β 因子的計算。若決定採用晶片內建看門狗，團隊必須意識到這可能會增加 β 因子，從而可能需要導入其他更強的安全機制（例如，增加設計多樣性、強化錯誤偵測碼）來補償，以確保最終產品能達到所需的 ASIL 等級。

### 公式 2：加總額外 β 因子

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | E10 |
| 公式原文 | `=SUM(E5:E8)` |
| 應用範圍 | E10, E26 |
| 計算值 | 0 |

**參照來源**：

此公式參照同一工作表中的 `E5` 至 `E8` 儲存格。這些儲存格是前一個公式（`=C*F`）的計算結果，分別代表不同耦合因素所貢獻的額外 β 因子。

**計算邏輯（白話文）**：

將 `E5`、`E6`、`E7`、`E8` 這四個儲存格的數值加總起來。

**物理意義**：

此計算結果代表由一組相關的共因耦合因素（例如，與晶片內建設計相關的所有因素）所產生的「總額外 β 因子」。它是對多個獨立風險貢獻的線性疊加，用以評估一類設計決策所帶來的整體 CCF 風險增量。

**標準依據**：

此加總行為遵循 **ISO 26262-5:2018, Annex D.4.3** 的原則，即 β 因子可以由多個獨立的耦合因素貢獻值相加而成。標準建議將所有識別出的耦合因素（CFi）的貢獻（Ci）進行加權和，以得出最終的 β 因子。此處的 `SUM` 函數即是該原則的直接應用。

**為什麼要這樣算**：

為了得到一個全面的、涵蓋多種潛在失效原因的總體風險評估值。單一的耦合因素（如共享電源）可能不足以構成嚴重威脅，但當多個因素（共享電源、物理鄰近、共享時脈）同時存在時，其累積效應可能會顯著提高共因失效的機率。透過將各個分項加總，安全工程師可以評估一組設計選擇（如將多個功能整合在單一晶片上）的總體影響，而不是孤立地看待每個決策。

**ASIC 部門注意事項**：

ASIC 團隊在進行設計時，應具備系統性思維。這個 `SUM` 公式提醒我們，設計中的各個部分並非獨立。一個看似無關的決定（例如，為了佈線方便而將兩個模組放在一起），可能會在安全分析中被識別為一個耦合因素，並貢獻一個非零的 β 因子分量。團隊在進行設計審查（Design Review）時，應主動與功能安全團隊合作，識別所有潛在的耦合因素，並理解它們將如何在這個加總公式中累積，影響最終的安全指標。

### 公式 3：計算最終晶片級 β 因子

| 項目 | 內容 |
|:-----|:-----|
| 儲存格 | E30 |
| 公式原文 | `=E28+E10-E26` |
| 應用範圍 | E30 |
| 計算值 | 33 |

**參照來源**：

此公式參照同一工作表中的 `E28`、`E10` 和 `E26` 儲存格。
- `E28`：根據上下文，這是一個基礎 β 因子值（此處為 33%），可能來自標準或公司內部的歷史數據，代表一個通用設計的基準 CCF 風險。
- `E10`：前一個公式的計算結果，代表由「增加」風險的設計決策所導致的總額外 β 因子。
- `E26`：與 `E10` 結構相同的另一個 `SUM` 公式結果，根據其在減法中的位置，代表由「降低」風險的設計決策（即安全機制）所帶來的 β 因子扣除項。

**計算邏輯（白話文）**：

從一個基礎 β 因子（`E28`）開始，加上因為特定設計選擇而增加的風險項（`E10`），再減去因為實施了某些安全機制而降低的風險項（`E26`），最終得到調整後的晶片級 β 因子（β_IC）。

**物理意義**：

此計算結果 `β_IC`（Resulting β-factor βIC）是針對當前這個特定 SSD Controller 晶片設計，最終估算出的共因失效（CCF）機率的量化指標。它不是一個通用的、標準化的數值，而是綜合考量了該晶片的基準風險、因特定設計（如片上冗餘）而增加的風險、以及因特定防護措施（如物理隔離）而減少的風險之後，一個客製化的、綜合性的評估結果。

**標準依據**：

這個「基礎值 + 增加項 - 減少項」的計算模型，是 **ISO 26262-5:2018, Annex D** 中 β 因子量化方法的實踐應用。標準提供了一個框架和一系列建議的權重因子，但最終的計算必須根據具體的硬體設計進行調整。`E28` 的基礎值可能來自於標準中的通用建議（例如，對於一個典型的 IC，β 因子可能在 1% 到 10% 之間），而 `E10` 和 `E26` 的計算則是對這個通用值的客製化調整。

**為什麼要這樣算**：

為了在標準化框架下，對一個具體的、獨特的設計進行精確的、可追溯的風險評估。功能安全分析的核心不是簡單地套用標準，而是將標準的原則應用於實際工程問題。這個公式完美地體現了這一點：它從一個公認的基準（`E28`）出發，然後根據本專案的具體設計細節（體現在 `E10` 和 `E26` 中）進行修正。這種方法使得分析結果既有理論依據，又能反映實際設計的優缺點，從而指導設計優化，例如，如果最終的 `β_IC` 過高，工程師就知道需要增加更多的防護措施（增加 `E26` 的值）或避免某些高風險設計（減小 `E10` 的值）。

**ASIC 部門注意事項**：

最終的 `β_IC` 值是衡量 ASIC 設計在共因失效方面穩健性的關鍵績效指標（KPI）。ASIC 團隊的每一個設計決策，都可能間接影響到 `E10` 或 `E26`，從而改變最終的 `β_IC`。團隊必須認識到：

- **設計的權衡**：增加功能或性能（可能增加 `E10`）與保證安全（需要增加 `E26` 的對應措施）之間存在權衡。例如，採用更先進但未經驗證的製程節點，可能會被視為一個增加風險的因素。
- **安全機制的有效性**：團隊不僅要實現安全機制，還要確保其有效性，並能提供證據來支持 `E26` 中的扣除項。例如，如果聲稱使用了物理隔離，就需要提供佈局圖（Layout Plot）來證明隔離的充分性。
- **與安全目標的關聯**：這個 `β_IC` 值將直接用於 PMHF 的計算。一個較高的 `β_IC` 會導致 PMHF 指標惡化，可能使得整個晶片無法滿足 ASIL B（PMHF < 100 FIT）或更高等级的要求。因此，控制 `β_IC` 是 ASIC 設計過程中與功能、性能、功耗同等重要的設計目標。
