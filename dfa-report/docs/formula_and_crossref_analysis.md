# 深度公式分析與跨頁關聯報告

## 一、公式總覽

本 Excel 共含三個有公式的 sheet，公式總數為 170 個。以下逐一分析每個公式的物理意義。

---

## 二、Sheet: Cut Set based Analysis — 公式解析

### 公式模式
所有 86 個公式均為同一模式：
```
P{row} = G{row} + I{row} + K{row} + M{row} + O{row}
```
（適用於 row 15 至 row 100）

### 物理意義
此公式計算的是 **Common Cause Criticality Index (CCCI)**，即「共因臨界指數」。CCCI 是五個評分維度的加總：

| 欄位 | 對應維度 | 評分範圍 | 物理意義 |
|------|---------|---------|---------|
| G 欄 | Spatial resolution（空間解析度） | 0-3 | 切割集元件之間的物理距離。距離越近，共因故障風險越高 |
| I 欄 | Sharing of common resources（共享資源） | 0-3 | 切割集元件是否共享電源、時脈等資源 |
| K 欄 | Conducting same information（傳導相同資訊） | 0-3 | 切割集元件是否傳導相同的訊號 |
| M 欄 | Same function and technology（相同功能與技術） | 0-3 | 切割集元件是否使用相同的功能/技術/製程 |
| O 欄 | Components in same package（同封裝元件） | 0-3 | 切割集元件是否位於同一封裝/外殼內 |

**CCCI = 空間距離分數 + 共享資源分數 + 相同訊號分數 + 相同技術分數 + 同封裝分數**

CCCI 值範圍為 0-15。值越高，表示切割集元件之間的共因故障潛力越大，需要更深入的調查或額外的緩解措施。

### 資料來源
切割集（Cut Set）來自 FTA（故障樹分析）或 ETA（事件樹分析）的輸出。每個切割集代表一組事件的 AND 組合，當所有事件同時發生時會違反安全目標。

### 隱藏欄位
S 欄被隱藏，包含評分選項的下拉清單值：`0, 1, 2, 3`（對應 S15:V15）。

### 條件格式
P15:P100 有條件格式化（可能是依 CCCI 值高低變色），G/I/K/M/O 欄也有條件格式化。

## 三、Sheet: CCF Calc. On-Chip Red. — 公式解析

此 sheet 計算的是 **β-factor（β因子）**，依據 IEC 61508 Ed. 2 標準，用於晶片內冗餘（On-Chip Redundancy）的共因故障量化。

### 公式群組 1：增加 β 因子的技術措施（Row 5-8）
```
E5 = C5 * F5    （Δβ = 基礎Δβ值 × 是否採用，0或1）
E6 = C6 * F6
E7 = C7 * F7
E8 = C8 * F8
E10 = SUM(E5:E8)  （增加的β因子總和）
```

| 行 | 技術措施 | Δβ值(%) | 物理意義 |
|----|---------|---------|---------|
| 5 | 晶片內看門狗作為監控元件 | 5 | 使用晶片內看門狗可能導致較高的 DC（診斷覆蓋率），但共因風險也增加 |
| 6 | 晶片內其他監控元件（如時脈監控） | 5 | 同上邏輯，晶片內監控增加共因耦合 |
| 7 | 獨立物理區塊間無交叉的內部連線 | 2 | 無交叉佈線降低短路風險 |
| 8 | 獨立物理區塊間有交叉的內部連線 | 4 | 有交叉佈線增加短路和共因風險 |

**物理意義**: 這些措施會**增加** β 因子（即增加共因故障的比例），因為它們代表晶片內部的耦合因素。

### 公式群組 2：降低 β 因子的技術措施（Row 14-24）
```
E14 = C14 * F14   到   E24 = C24 * F24
E26 = SUM(E14:E24)  （降低的β因子總和）
```

| 行 | 技術措施 | Δβ值(%) | 物理意義 |
|----|---------|---------|---------|
| 14 | 1a: 不同通道的多樣化故障控制措施 | 4 | 多樣性降低共因風險 |
| 15 | 1b: 功能多樣性+不同通道故障控制 | 6 | 更高程度的多樣性 |
| 16 | 2: EMC 測試（含額外安全裕度） | 5 | 電磁相容性測試降低環境共因 |
| 17 | 3: 各區塊獨立電源引腳 | 6 | 電源隔離降低共因 |
| 19 | 5: 獨立區塊間的接地引腳 | 2 | 接地隔離 |
| 20 | 6a: 高診斷覆蓋率 DC≥99% | 7 | 高 DC 快速偵測故障 |
| 21 | 6b: 區塊間溫度感測器（低效能） | 2 | 溫度監控（無診斷） |
| 22 | 6c: 區塊間溫度感測器（高效能） | 9 | 溫度監控（有診斷） |
| 23 | 6d: 故障影響分析/測試 | 9 | 系統性分析 |
| 24 | 6e: 監控電路在高溫下仍可運作 | 7 | 極端條件下的可靠性 |

### 最終公式
```
E28 = 33                        （基礎β因子 βB-IC = 33%）
E30 = E28 + E10 - E26           （最終β因子 βIC = 基礎 + 增加 - 降低）
```

**物理意義**: 
- **βB-IC = 33%** 是 IEC 61508 定義的晶片內冗餘基礎β因子，意味著在沒有任何措施的情況下，33% 的故障被認為是共因故障。
- **βIC = βB-IC + Σ(增加因子) - Σ(降低因子)** 是最終的β因子。
- 目前所有措施的 F 欄（是否採用）均為 0，所以 βIC = 33 + 0 - 0 = 33%。
- 此值用於 ISO 26262 的定量安全分析中，計算共因故障對 SPFM、LFM、PMHF 指標的貢獻。

## 四、Sheet: CCF Calc. Logic — 公式解析

此 sheet 計算的是 **β-factor for Logic subsystem**，依據 IEC 61508 Ed. 2 的 Annex D。

### 欄位結構
| 欄位 | 含義 | 說明 |
|------|------|------|
| B | XLS | 邏輯子系統的 X 分數 |
| C | YLS | 邏輯子系統的 Y 分數 |
| D | XSF | 感測器/最終元件的 X 分數（隱藏欄） |
| E | YSF | 感測器/最終元件的 Y 分數 |
| F | X | 實際 X 得分 = B×H 或 D×H |
| G | Y | 實際 Y 得分 = C×H 或 E×H |
| H | yes=1 no=0 | 是否採用此措施 |

### 公式模式
```
F{row} = B{row} * H{row}    （X 得分 = XLS基礎分 × 是否採用）
G{row} = C{row} * H{row}    （Y 得分 = YLS基礎分 × 是否採用）
```

### 六大評估類別

**1. Separation/Segregation（分離/隔離）** — Row 5-9
評估通道間的物理分離程度（信號線分離、PCB 分離、硬體單元分離等）

**2. Diversity/Redundancy（多樣性/冗餘）** — Row 11-20
評估通道間的技術多樣性（不同電子技術、不同設計者、不同校準方法等）

**3. Complexity/Design/Application/Maturity/Experience（複雜度/設計/應用/成熟度/經驗）** — Row 22-27
評估設計的成熟度和複雜度（跨通道連接、現場經驗、保守額定等）

**4. Assessment/Analysis and Feedback（評估/分析與回饋）** — Row 29-31
評估分析活動的充分性（FMEA/FTA 結果、設計審查、現場故障分析）

**5. Procedures/Human Interface（程序/人機介面）** — Row 33-39
評估維護和診斷程序的品質

**6. Competence/Training/Safety Culture（能力/培訓/安全文化）** — Row 42-43
評估人員培訓的充分性

**7. Environmental Control（環境控制）** — Row 45-47
評估環境保護措施

**8. Environmental Testing（環境測試）** — Row 49
評估環境免疫測試的充分性

### 匯總公式
```
B51 = SUM(B5:B49) = 62.5     （XLS 最大可能分數）
C51 = SUM(C5:C49) = 50       （YLS 最大可能分數）
D51 = SUM(D5:D49) = 55       （XSF 最大可能分數）
E51 = SUM(E5:E49) = 46.5     （YSF 最大可能分數）
F51 = SUM(F5:F49) = 0        （實際 X 得分）
G51 = SUM(G5:G49) = 0        （實際 Y 得分）
H51 = SUM(H5:H49) = 0        （採用措施數量）
```

### β 因子計算公式
```
B53 = F51 + G51               （Score S = x + y）
B55 = [使用者輸入的 Z 值]      （Z 值來自 IEC 61508 Table D.2/D.3，基於診斷覆蓋率和測試間隔）
B57 = F51 * (B55 + 1) + G51   （Score SD = x × (z + 1) + y）
B59 = IF(B53<45, 5, IF(B53<70, 2, IF(B53<120, 1, 0.5)))    （β 值查表）
B61 = IF(B57<45, 5, IF(B57<70, 2, IF(B57<120, 1, 0.5)))    （βD 值查表）
```

### 物理意義

**Score S** 是不考慮診斷的基本分數，**Score SD** 是考慮診斷後的調整分數。Z 值反映了診斷覆蓋率和測試間隔的效果。

**β 和 βD 的對照表**:

| Score 範圍 | β 或 βD 值 (%) |
|-----------|---------------|
| < 45 | 5% |
| 45-69 | 2% |
| 70-119 | 1% |
| ≥ 120 | 0.5% |

目前所有措施均未啟用（H=0），所以 S=0, SD=0，β=βD=5%（最高風險等級）。

### 右側參考表（N-R 欄）
包含 IEC 61508 的三個查表：
- Table D.2: Z 值（可程式電子元件），基於診斷覆蓋率(60%/90%/99%)和測試間隔(<1min/1-5min/>5min)
- Table D.3: Z 值（感測器或最終元件），基於診斷覆蓋率和測試間隔(<2h/2h-2d/2d-1w/>1w)
- Table D.4: S/SD 分數對應的 β/βD 值
